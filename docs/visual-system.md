# Visual System

The AI-brief visual system should feel like a premium Chinese-first AI intelligence publication with practical product surfaces.

## Direction

- Editorial but operational.
- High whitespace, not dense dashboard.
- Strong content hierarchy.
- Image-forward cards.
- Calm surfaces and high legibility.
- Practical, not cyberpunk.

## Typography

Use the current Geist-based direction unless a later task changes typography.

Rules:

- Large display type only for true hero sections.
- Compact titles in cards.
- Monospace for dates, scores, and metrics.
- Avoid negative letter spacing.
- Chinese text must wrap cleanly on mobile.

## Color

Use restrained color:

- Background: off-white or very light neutral.
- Text: near-black, not pure black.
- Card: white or warm paper.
- Border: soft neutral gray.
- Accent: one calm primary accent.
- Status chips: subtle backgrounds, not saturated badges.

Avoid:

- Purple-blue AI gradients as the dominant identity.
- Neon glow.
- One-note beige or dark slate systems.
- Decorative gradient orbs.

## Layout

Desktop:

- Max width around 1120px-1280px for content pages.
- Wider editorial heroes can use asymmetric split layouts.
- Directories can use responsive grids with filters above the grid.

Mobile:

- Single column.
- No horizontal overflow.
- Image cards keep stable aspect ratios.
- Long Chinese titles wrap intentionally.
- Nav should not block the first content interaction.

## Cards

Cards must help the user decide within 5 seconds.

Every content card should expose:

- Title.
- One-sentence takeaway.
- Why it matters.
- Content type.
- Audience.
- Reading time.
- Action label.
- Impact and confidence.
- Source.
- Date.
- Thumbnail or cover image when available.

Cards should not become long summaries. Move deeper analysis to detail pages.

## Images

Images are product assets, not decoration.

Use images for:

- Daily issue cover.
- Development blocks.
- Tools and project thumbnails.
- Playbook covers.
- Model radar visuals.
- Research/article concept covers.

Image requirements:

- Every image has alt text.
- Every generated image has prompt metadata.
- Every non-generated image has credit/source metadata.
- Public pages only show approved images.
- Generated images should avoid readable text by default.

## UI States

Every user-facing list or page should support:

- Loading state.
- Empty state.
- Error state.

Loading states should use layout-matched skeletons. Empty states should tell the user what will appear there, not just say "no data."

## Interaction

Keep motion restrained:

- Buttons should have tactile active states.
- Directory filters should respond instantly.
- Follow-along Playbook steps can show completion states later.

Do not introduce heavy animation dependencies before MVP.

## Explicitly Out Of Scope

- Email subscription form.
- Subscribe CTA.
- Login CTA.
- Paid membership UI.
- Reader-count social proof.
