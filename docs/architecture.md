# architecture.md

Last updated: 2026-05-15

## Purpose

This document is the long-term architecture memory for AI Brief. Keep it current whenever the system flow, data contracts, agent pipeline, or frontend surfaces change.

AI Brief is a Chinese-first AI intelligence product built around:

```text
Information -> Judgment -> Action
```

The current MVP is a Vite + React + TypeScript app backed by static JSON data in `public/data/`. Dynamic ingestion exists for GitHub Trending projects; model and academic article archives are curated datasets.

## System Structure

```text
Browser
  -> Vite React app
      -> hash router in src/App.tsx
      -> pages in src/pages/
      -> static data loaders in src/lib/data.ts
      -> typed data contracts in src/types.ts
  -> public/data/*.json
      -> trending.json   generated by ingest pipeline
      -> models.json     curated company/model archive
      -> articles.json   curated academic paper version archive

Dev server
  -> vite.config.js
      -> POST /__ingest
      -> streams scripts/ingest.mjs output

Ingestion
  -> scripts/maybe-ingest.mjs
  -> scripts/ingest.mjs
  -> .cache/analyses.json
  -> public/data/trending.json
```

## Active Product Surfaces

### Home

`#/`

Home is a navigation hub only. It should not hold full product content. It links to News, Models, Projects, Skills, and Articles.

### Projects

`#/projects`

Shows GitHub Trending daily / weekly / monthly boards. Data comes from `public/data/trending.json`.

Project detail:

```text
#/repo/:owner/:name
```

Project detail includes README quick read, key concepts, architecture flow, novelty, ecosystem, limitations, try-it steps, and scoring.

### Models

```text
#/models
#/models/:companyId
```

Curated company-level model evolution archive. Current companies include DeepSeek, OpenAI, Anthropic, Google DeepMind, and Meta.

Each model company should explain:

- each model generation on its own;
- architecture / design intent;
- training or post-training direction;
- benchmark signals;
- limitations;
- relation to the next generation;
- product/API updates around the model line.

### Articles

```text
#/articles
#/articles/:paperId
```

Curated academic paper version analysis. Current examples include Transformer, RAG, LoRA, and DPO.

Each paper should explain:

- arXiv version timeline;
- what changed in each version;
- why those changes matter;
- method, experiments, limitations;
- professor lens;
- verification checklist for the student.

### Placeholder Sections

News and Skills are still placeholders. Courses has been removed from product scope. Do not pretend removed or placeholder sections are implemented.

## Agent Flow

There is no separate `agents/` directory yet. The current agent-like flow lives inside `scripts/ingest.mjs`.

### Project Ingestion Agent Flow

```text
GitHub Trending HTML
  -> parse repository cards
  -> GitHub API README fetch
  -> light DeepSeek analysis for every repo
      -> tldr
      -> tags
      -> light summary
      -> worthDeepDive score
  -> candidate selection
      -> worthDeepDive >= threshold
      -> cap per run
  -> deep DeepSeek analysis for selected repos
      -> atGlance
      -> whyItMatters
      -> keyConcepts
      -> howItWorks
      -> novelty
      -> ecosystem
      -> limitations
      -> tryIt
      -> score breakdown
  -> write public/data/trending.json
```

### Model Archive Flow

```text
Official model sources
  -> manual research
  -> structured model data in public/data/models.json
  -> scripts/validate-models.mjs
  -> Models UI
```

This flow is curated instead of ingested. It should prioritize high-signal official sources and stable teaching value over daily churn.

### Academic Article Flow

```text
arXiv / official paper pages
  -> manual research
  -> structured paper data in public/data/articles.json
  -> scripts/validate-articles.mjs
  -> Articles UI
```

This flow is curated. It should focus on paper versions, method changes, evidence, limitations, and student verification.

## Pipeline

### Local Dev

```bash
npm install
npm run dev
```

`npm run dev` runs `predev`, which calls `scripts/maybe-ingest.mjs`. If `public/data/trending.json` is stale, ingestion may run before Vite starts.

### Ingestion

```bash
npm run ingest
npm run ingest:force
npm run ingest:dry
```

Important defaults:

- `--limit=15`
- `--cap=6`
- `--worth=60`
- cache file: `.cache/analyses.json`
- cache TTL: 7 days

### Validation

```bash
npm run validate
npm run lint
npm run typecheck
npm run build
```

Validation currently covers:

- `public/data/trending.json`
- `public/data/models.json`
- model workbench UI presence
- `public/data/articles.json`

## Data Contracts

The canonical TypeScript contracts live in:

```text
src/types.ts
```

Do not hard-code mock content inside UI components. Add structured data to `public/data/*.json` and update validation scripts when the contract changes.

## Current Architecture Risks

- Static curated data is easy to edit but can become large and stale.
- GitHub Trending ingestion depends on HTML parsing, which can break if GitHub markup changes.
- DeepSeek output is JSON-constrained by prompt and post-validation, but hallucination risk remains.
- No shared content evaluation service exists yet; evaluation logic is embedded in prompts and validation scripts.
- No real database exists yet; all public data is file-based.
