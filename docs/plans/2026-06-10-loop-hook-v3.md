# Loop Hook v3 — 合并双 hook + 句级分类 + 契约库 + Stop 闭环

> 2026-06-10。Kevin 授权（"按照你推荐的来改进，你持续改进至完成 loop engineering"）。
> 协议语义同步升级：7 格展示 + 断层提问 → **不等"对"，自主推进到可运行交付**（红线 🔴 仍停）。

## 问题（实证）
1. **关键词分类脆弱**：0610 两类实证——假阳（系统通知/疑问句布防，已打补丁）+ 假阴（"按推荐来改进/重写/更改"无一命中词表，实施请求不布防）。问句判断只看整条消息结尾。
2. **双分类源不一致**：loop-protocol.mjs（模型判"实质事"）与 loop_contract_gate.py（关键词判"实施类"）对同一消息可给出相反判断。
3. **再布防 churn**：`.agent/archive/` 0610 一天 22 份契约，20:02 五秒内 3 份——每条实施 prompt 重布防+归档，半重复任务反复填契约（违背 "skills compound, prompts burn"）。
4. **无闭环**：协议要求走完更新 memory + 6 问汇报，无机器检查。
5. **陈旧布防**：armed 状态跨 turn 残留会拦住后续无关写入（项目级共享 state 事故根因）。

## 改法
- **合并**：删 loop-protocol.mjs；gate 脚本在 UserPromptSubmit 统一发提醒（未布防=1 行轻提醒；布防=完整 7 格 v3 文案）。单一分类源。
- **句级分类**：按句切分；实施词（EN 词边界正则 + ZH 扩词表：改进/优化/重写/升级/补全/实施/落地/更改）只在**非疑问句**内才算；疑问句= 句尾 ?/？/吗/呢 或句内 是否/是不是/要不要/怎么/如何/为什么/哪些 等。保守偏向：宁假阴不假阳。
- **契约库** `.agent/contracts/<type>.md`：过审 loop 类型存库（含 `Reuse triggers:` 触发词行）；布防时命中触发词 → 复制+换 hash → 验证即过 → 免重复填。库由 Claude 手动晋升（一个 loop 类型跑稳后归库）。
- **冷却 180s**：state 创建 3 分钟内的新实施 prompt 不重布防（视为同任务延续）。
- **Stop hook**：① armed 未实施 → 自动拆防（修 #5）；② 开闸过且未闭环 → block 一次提醒（6 问汇报 + memory 更新），置 closed 后第二次 Stop 放行（保证不死循环）。
