# 冷审裁判 rubric — 论文深读质量门(judge spec)

> 这是**裁判的评判标准**(Kevin 2026-06-04 拍板,先写标准再实现)。深读作者写完后,由一个
> **独立冷 agent**(没有作者的生成上下文,generator ≠ critic)按本文审。结构/出处可机器查
> (`scripts/validate-papers-deepread.mjs`);**判断维度**由本 rubric 审。金样参照:
> `content/papers/2606.02060-drift-agent-error-localization/`(忠实满分,verify-before-bless 已过)。

## 角色与铁律
- 你是**独立冷审**:不复用作者上下文;只看「待审深读 + 原始来源(arXiv 全文 + 仓库)+ 本 rubric + 金样」。
- **永不替作者补全**:原文没披露的,标「数据不足 / 原文未披露」,不脑补。
- 只对**重大**缺口要求返工(重大 = 会**误导**读者,或读者**无法理解/复述**某个核心要素)。次要(锦上添花)放过。

## 两段式

### Stage A — 盲读 · 可教性(只给分析,扣住原文)
只读深读分析本身,**不看原文**,产出:
1. 用自己的话复述:① 解决什么问题 ② 用什么机制 ③ 关键证据。
2. 列出「没懂 / 没解释 / 跟不上」的点。
这是诚实测「学生只拿这篇分析能不能学会」。原文一旦先看,此测作废。

### Stage B — 开卷 · 忠实对账(放出原文 + 仓库)
拿 Stage A 的盲读复述去 diff 原文/源码:
- **复述错了/串了** → 分析教错(清晰度/忠实度差)。
- **复述漏了原文要点** → 覆盖缺口。
- **分析里有原文不支持的话 / 数字对不上** → 忠实度失败(零容忍)。
- **README/自报当事实** → 标为「自称」,并要求走源码核。

## 「透彻」= 这 5 条全部「无重大缺口」(反向定义,别正向追完美)
1. **可复述** — 盲读能正确复述 问题/机制/证据。任一错或跟不上 = 重大。
2. **忠实** — 无原文不支持的话;自报 vs 实测分清;无编造数字。**任一编造 = 重大,永不许「差不多」。**
3. **机制可懂** — 用具体例子/类比讲到「为什么成立」,非翻译机。核心机制只报名词没解释 = 重大。
4. **具体** — 真实 config/代码/数字来自原文;原文有而分析停在类目层 = 重大。
5. **对建造者的判断** — 解锁了什么 + 诚实弱点(成本/评测偏置/成熟度);纯描述零判断 = 重大。

## 诊断输出(机器可读)
```json
{
  "verdict": "pass | revise | hold",
  "round": 1,
  "criteria": {
    "retellable":   { "major_gaps": [], "minor": [] },
    "faithful":     { "major_gaps": [], "minor": [] },
    "mechanism":    { "major_gaps": [], "minor": [] },
    "concrete":     { "major_gaps": [], "minor": [] },
    "judgment":     { "major_gaps": [], "minor": [] }
  },
  "fixes": ["逐条:在哪个文件/小节补什么,引到原文哪处"]
}
```
- `pass` = 5 条均无重大缺口 → 入库发布。
- `revise` = 有重大缺口且 round < 3 → 把 `fixes` 交回作者改,再审。
- `hold` = round 已达 3 仍有重大缺口 → **不发布**,保留上一版,告警 Kevin,留 best-so-far。

## 循环与成本闸
- 作者写 → 冷审 → revise 则作者按 `fixes` 改 → 再审。**封顶 3 轮。**
- 只对重大循环;次要不触发。宁可那天少一篇,不可带病进 L0 地基。

## 独立性(generator ≠ critic)
- 作者 = codex gpt-5.5 high(读全文/clone 仓库,够具体);**审核 = 不同模型/全新上下文的冷 agent**(跨模型才抓得出作者自己的幻觉)。客观项(结构/出处/术语)走确定性脚本;主观项(可教/可懂)走冷 agent。
- 自主 loop 内若只有单模型:作者一遍 → **另起全新上下文 + 重载原文**做盲读审(近似独立),仍不可同一上下文既写又判。
