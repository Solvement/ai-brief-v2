# 项目栏大重做 spec（Kevin 2026-06-08 Goal）

> 起因:Kevin「项目深度分析做得非常差,差到不想看」。重做三轴:选品砍到精英、每个本地深析才上、呈现满宽+少字+图。决策:**~10-12 个精英**;交叉验证源**Claude 选一组**(下列)。

## 总原则
- **少而精**:GitHub 上只留**高 star + star 增长快**的;拿不准就**跨站交叉验证,多源同现=好**;其余一律不上。
- **每个项目都先本地深析才上**(codex/claude clone+读源),不是模板卡。
- **呈现**:充分利用屏幕宽度;**去掉大段英文原文**;**别堆大量文字**;**架构用 Mermaid 图**表达。

## 轴一 · 选品(砍到 ~10-12 精英)
- 主源 GitHub:星绝对值 + **月增 star 速度**双高(硬门槛,codex 定具体数值,偏严,宁缺毋滥)。
- **跨站交叉验证**(判"够不够好"):① GitHub trending 多窗同现;② Hacker News(Algolia API:repo 的 github.com URL 有高分 story = 社区验证);③ 第三方 GitHub 榜(OSSInsight ossinsight.io / Trendshift trendshift.io,可抓即用)。
- **精英判定** = (高 star ∨ 月增很快) ∧ 在 **≥2 源**出现。只有精英进栏,**其余全砍**(不再有 list_only 满屏卡)。
- 顺手根治 [[proj-deep-render-gap]] 暴露的**榜单累积 bug**:`makeBoard` 不截断 + `mergeRepos` union 累积旧 windows → 日榜 78。精英化后榜单本就小;同时给 makeBoard 加上限 + 只显示当前在榜(starsGained>0 / 当前 rank),不显示陈旧残留。

## 轴二 · 分析范式(废掉重写——这是核心,现在那版"差到不想看")
每个精英项目本地深析(clone+读源),产出**新范式**,硬要求:
- **少文字**:不堆大段;每段就几句应用判断。
- **去英文原文**:不许整段贴英文 README/源码;要中文转述 + 必要时极短代码/命令片段(带中文说明)。
- **架构用图**:核心架构/数据流/Agent loop 用 **Mermaid 图**表达,不用长段文字描述。
- **应用判断为主**(承袭 [[user-goal-applied-ai-builder]]):它是什么(中文一句话)/解决什么/怎么用/什么时候用/**横向对比 ≥2 个点名同类**/成熟度+风险。不堆机制数学。
- 字段精简:一句话定位 / 架构图(mermaid) / 核心能力(≤3 bullet) / 怎么跑(install+最小例,极短) / 横向对比 / 对你意味着什么 / 风险。删掉现有那些空泛长字段。

## 轴三 · 前端(满宽 + 图 + 少字)
- 项目深读页:**满宽布局**(像论文页那次满宽改造),Mermaid 图大图懒加载渲染,中文为主,信息密度适中不疲劳。
- 列表卡:作者/标题/star/增长 chip 之外,**中文一句话说明这个项目是关于什么**(轴二的"一句话定位")。只放精英 ~10-12 张。
