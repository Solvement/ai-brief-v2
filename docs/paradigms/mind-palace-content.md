# Mind Palace 内容契约——精读为记忆宫殿产什么（canonical，2026-06-12 Kevin 定义）

> 从 mind palace 反推精读内容：不是"写篇好文章"，是"喂饱记忆宫殿"。论文范式（密度分区/类比讲机制/discovery_trace）与项目范式 v3（白板测试）都受此契约约束。

## 每篇精读必产三要件

### 1. 单独分析（单项视角）
白板测试 eval：读完不看原项目/论文，能画出**结构、优势区间、自己的理解、创新、解决了什么问题、用了什么方法**。
- 蒸馏面：problem_solved / method / result / innovation / weakness（≤15% 篇幅）/ core_concepts（承重概念，连边锚）/ discovery_trace（论文侧：怎么找到解法，禁编造）。
- 生成法：导师式（先判断该项目的灵魂角度），禁填表式。

### 2. 群体分析（谱系视角）——Kevin 2026-06-12 定义，同日晚最终锁定
**同一个东西，单独看 vs 放进同类项目+论文里看，理解不一样；两个都做才完整理解问题本质。** 群体分析是几篇论文+项目连成的「小神经元回路」涌现出的、比单篇之和更多的东西——是外脑区别于"关系查看器"的地方。

**A. 单篇侧：`group_position`**（每篇精读必带）：放进同 track 谱系（项目+论文混排）——它站哪个位置？相对取舍（牺牲什么换什么）？补了哪块空白？谁先谁后（演进）？写群体定位时发现的关系（取长补短/可合并/演进自）= typed 边的来源；写不出 = 该 track 蒸馏太浅。

**B. 簇侧：簇综述（cluster synthesis）**——每个有组织的连通簇（≥3 成员/track）产一份，入 `public/data/brief/cluster-synthesis.json`，前端星图放大面板渲染。簇综述含以下层（Kevin 锁定，顺序≈面板顺序）：
1. **回路主题**：这一簇是哪个主题/回路（一句话定性 + 成员名单）。
2. **文章↔文章关系**：typed 边 + 一行缘由（取长补短/对比/张力/谱系/互补/替代/互证/评测…）。
3. **方法论对比**：每个成员的方法立场 + 彼此差在哪条轴。
4. **分别 / 共同解决什么问题**：各自切一块 vs 合起来覆盖的问题空间 + 还没覆盖的缺口。
5. **核心概念 + 方法提取**（Kevin 修正：不做空泛"任意问题迁移"，要 grounded）：按「用 **M+N+X 方法** 解决了 **Y 问题**，展示了 **Z 结果**，现在是 **a 阶段**」的格式，提取每条线的核心概念与方法。
6. **比单篇多读出的理解**：放一起才见的那条轴 / 已收敛 / 还开放——单看一篇看不到。
7. **演进 & 前沿**：谱系时间线、当前最强/SOTA 在哪、谁被取代弃用。
8. **簇级自用判定（self-evo）**：整个回路对*我们自己系统*意味着什么改动。⚠**改进/替换要谨慎**（Kevin 红线）：后端是**优化不是增加**，不能有新方法就往上堆（渺小前端撑不起臃肿后端）；**不硬套**，没真用处就如实标 `no`。
9. **置信度 + 客观原因**：这套理解多稳？**客观分析**为什么确信/不确信、什么新论文会推翻这条轴（不空给一个分数）。
10. **缺成员 → 反哺选稿**：这回路还缺哪个*已知但没深读*的邻居才完整 → 指向"该深读谁"（接缺口视图，闭环选品）。
11. **类科研·新假设**：从这簇长出的想试的实验，**必标「假设，非已验证结论」+ grounded（有依据）/ speculative（脑洞）**——防"类科研"滑成幻觉吹牛（同无编造红线）。
- **后置（先不做）**：跨簇桥——这回路和别的回路在哪触碰（最强的"更多理解"常在接缝，但先把单簇做好）。

**贯穿红线（Kevin 2026-06-12）：后端是优化不是增加，前端跟着后端进化。** 簇综述每多一层后端字段，前端面板必须**同步渲染**那一层；不留前端不展示的孤儿后端数据，不堆叠未消化的方法。这条管着整个 build。

### 3. 自用判定（self-evo 必经步骤，非可选贴纸）
**每篇精读完成时必须回答：这能不能用到我们自己系统上？**（Kevin："你做完每一个精读都要判断自己是不是能用上。如果能用上就去用。"）
- 判定三态：`apply`（非红线 → 直接应用 → verify 全绿 → 汇报"从哪学的/改了什么/怎么证明没破坏"，走 L1 授权书）/ `queue`（红线或大改 → 进 self-evo-queue 周审）/ `no`（写一句为什么用不上——"收敛也是有效结果"）。
- 禁"描述性便利贴"：self_evo_use 写了"可以改进 X"却无人执行 = 违约。判定结论必须落在精读产物的 `self_evo_verdict` 字段。

## 验收（冷审用同一把尺）
- 白板测试：冷审 agent 只读分析，试画白板——画不出结构/优势/创新 = FAIL。
- 群体测试：冷审读 group_position，问"它和同类的相对取舍是什么"——答不出 = FAIL。
- 自用测试：self_evo_verdict 三态必有其一且有理由。
- 簇综述测试：11 层齐（回路主题/关系/方法论对比/问题地图/M+N+X 提取/多读理解/演进前沿/簇级自用/置信度+客观原因/缺成员/新假设）；新假设带「假设非结论」+ grounded/speculative 标；簇级自用不硬套（无真用处标 no）；置信度带客观原因不空给分数。
- **前后端同步门（Kevin 红线）**：簇综述每一层后端字段在前端星图放大面板有对应渲染——有孤儿后端字段（前端不展示）= FAIL。后端是优化不是增加。
