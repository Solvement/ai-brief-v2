### 02:31-06:31 从腾讯、创业、剪映、抖音、B站到动画学习

### 06:31-12:30 OeOe 的起心动念、Agent 形态、早期链路与 Sora2

**这段在谈什么 / 主持人问了什么**

主持人把话题转向 OeOe：什么时候有起心动念，什么时候看到切入口，做 OeOe 是否受到其他 AI 产品启发，模型能力解锁对 OeOe 有什么影响。

**嘉宾的回答（带时间戳）**

- Nono 说做动画是从大学就开始想做但没机会做；做 OeOe 的起点是在字节工作时，看到“二年图像生成”的 Demo / 输出后，第一时间想到“这个可以做动画”。（06:31-07:02）
- 她说之后无论去 B 站了解动画行业，还是之前做“离谱”，都是在试探怎么切入动画行业；真正创业时才找到切入口。（07:02-07:31）
- 关于切入口，她说有被动和主动两面：被动是前一个环境下很难再做下去，必须出来自己做；主动是多模态模型开始“卷起来”，类似以前语言模型开始卷起来，是一个好时机。（07:31-08:01）
- 受 AI 产品启发方面，她说今年上半年 Agent 开始火，团队在想怎么切入动画时发现 Agent 是“非常完美的形态”：可以调用各种模型；动画制作本身是流水线式制作，需要各种职业配合完成成品；Agent 相比传统 GUI，给创作者的自由度更高，不是把功能无限叠加。（08:01-08:56）
- 她说团队 7 月规划、8 月开始做，第一个版本路线和很多视频 Agent 类似。她把当时 AI 视频路线分成两类：“多参宗”（多张图片做参考图）和“首尾帧宗”。团队一开始采用首尾帧方式，因为它在单个镜头里的稳定性更好。（08:56-09:42）
- 她解释早期链路：如果一共 8 个镜头，就做 9 张图，用首尾帧连续衔接，再生成一段段小视频拼起来。（09:42-10:01）
- 她说当时的创新点是用 Task Agent 判断每个分镜需要表达什么、每个模型的优缺点是什么，再自动把不同模型配到相应镜头。例如武打镜头用擅长武打的模型，人物情绪镜头用擅长人物情绪的模型。（10:01-10:31）
- 她说这个版本做了一个月后效果已经较好，正在加 TTS、背景音乐和音效时，Sora2 出来了。她看到有人用 Sora2 做动画，觉得“完全看不出来任何 AI 的痕迹”，像完整动画，于是团队决定接 Sora2。（10:31-10:59）
- 第一个接完 Sora2 的视频是研发做的“小螃蟹和一个小星星在那打篮球”，她说自己看到后非常兴奋，第一印象是：“不会这事真的让我们做成了吧。”（10:59-11:20）
- 主持人问螃蟹和海星打篮球是不是一句直出。Nono 说是“一句直出”，不过之前首尾帧链路也是自己出的，也是一句直出，只是生成方式不同。（11:20-11:42）
- 她说 Sora2 的优势在于表现力和切镜，“镜头镜头之间的切是非常自然的”，有很多蒙太奇手法；不足在于它是参考图加文本，文本权重很大，导致一致性不那么稳定。相比之下，首尾帧稳定性更好，因为是从那一帧开始推演。（11:42-12:14）
- Sora2 出来后，团队本来准备两条链路同时跑，后来决策“老链路先不做了，全部切到 Sora2”，因为虽然有问题，但效果非常棒。（12:14-12:30）

**例子或论据原样呈现**

- 8 个镜头做 9 张图，首尾帧一段段生成再拼起来。（09:42-10:01）
- Task Agent 根据镜头类型分配模型：武打、人物情绪等。（10:01-10:31）
- 小螃蟹和小星星打篮球，是研发做的第一个接 Sora2 后的视频。（10:59-11:20）

### 12:30-18:31 Sora2 会不会吞掉 Agent；大模型像菜场，Agent 像餐馆

**这段在谈什么 / 主持人问了什么**

主持人问：如果未来 Sora4、Sora5 能端到端做得更好，OeOe 这样的产品会不会被吞掉；OeOe 与 Sora2 直接使用相比做了哪些不同。

**嘉宾的回答（带时间戳）**

- Nono 说不担心未来 Sora 端到端吞掉 Agent，反而觉得视频模型对 Agent 类产品“非常友好”。（12:30-13:03）
- 她认为视频模型不太可能“大一统”，因为数据标准、数据、数据质量不同，吐出来的东西各有特色。她把各种模型比作“大型超市”或“大型菜场”，是做菜的“料”；Agent 则像用这些料开餐馆。OeOe 现在像“一家川菜馆”，要去各个菜场挑适合自己人群的食材。（13:03-13:52）
- 她说 Agent 的主要工作不只是拿模型，团队大概 60% 的工作是在“调这些很细微”的东西，用户在产品上看不到。她继续用比喻说：用户也可以自己去超市买菜做饭，也可以下馆子吃现成的；视频 Agent 也会有川菜馆、粤菜馆、香菜馆、火锅等不同形态，大家一起把“小吃街”做起来。（13:52-14:44）
- 主持人追问 OeOe 相比 Sora2 直接一句话出片做了什么不同。Nono 以 MV 为例说，用相同 Prompt 和相同图片，OeOe 放到 Sora2 里也达不到 OeOe 的水平，因为 OeOe 会把用户一句话扩写；团队找了很多二次元 MV 做得好的视频，用自己的方法让模型学习，形成知识库。用户写 Prompt 时，会调这个知识库，用文本方式吐出 Prompt，搭配镜头语言和音效生成，一键输出。（15:01-16:01）
- 她说这里有大量工作在于“知识库应该如何建立”。（15:30-16:01）
- 她又举剧情分镜例子：用户常说分镜之间连贯性不好，但这也是团队的一种选择。如果每个分镜特别连贯，剧情会“非常平”，缺少突然转折和起伏，所以要不停地调到合适状态。（16:01-16:31）
- 关于场景一致性，她说用户反馈场景一致性不好，因为 Sora2 文本权重很重，团队喂给 Sora2 的内容里没有场景图，只在 Prompt 描述场景，导致不同分镜场景可能不同。后来为了让场景一致，喂了场景图，但又“太一致”，画面死板，场景和人物贴合像贴片。她用餐馆比喻：用户说川菜馆不辣，于是团队使劲放辣子，结果又太辣。（16:31-17:51）
- 主持人继续问如果未来 Sora2 解决一致性，甚至掌握二次元 MV 等垂直知识怎么办。Nono 仍用超市熟食和餐馆比喻：超市可以用自己的料做熟食，但熟食和餐馆不一样，餐馆仍有自己专注的人群和专业性，这种东西“无法穷尽”。（17:51-18:31）

**例子或论据原样呈现**

- 大模型是大型菜场 / 超市，Agent 是餐馆，OeOe 是川菜馆。（13:03-14:44）
- OeOe 找二次元 MV 视频，形成知识库，用来扩写用户 Prompt，并搭配镜头语言、音效。（15:01-16:01）
- 分镜不完全连贯是为了避免剧情太平、缺少起伏。（16:01-16:31）
- 场景图喂得太重会让场景一致但画面死板，人物像贴片。（16:31-17:51）

### 18:31-24:01 OeOe 的目标用户、意外用户和场景减法

**这段在谈什么 / 主持人问了什么**

主持人问 OeOe 目前的目标用户是谁，以及内测中是否出现意外用户；后续会不会对用户做减法。

**嘉宾的回答（带时间戳）**

- Nono 说团队一开始考虑过切入“漫剧”赛道，但发现漫剧还没有 ready：第一，比较考验剧本，而模型和团队在剧本上不是非常有优势；第二，漫剧是非常投流的生意；第三，漫剧在成熟工作流里是效率优化，仍非常依赖人力，而团队不想做非常依赖人力的事情。（18:31-19:39）
- 她也解释为什么不做类似“离谱”的 UGC：UGC 内容消费属性太短、信息量不足，难以支撑大型 App 转起来；UGC 需要很大用户体量，投入成本和推理成本都高，所以内容社区产品很难发展得非常大。（19:39-20:30）
- 团队想切的是自媒体人：一个人或小工作室，内容适合用动画呈现。她分了几类：本身做动画、以 IP 为出发点持续更新的人；ACG 里做 MV 类型的人；以及不做动画但讲历史、讲科普、适合用动画展示的人。（20:30-21:52）
- 对本身做动画的自媒体，她说调研发现两三个人的小工作室现在大概一周更新一集；用 OeOe 基本一天可以更新十几集，如果精挑细选，一天更新一到两集完全没问题，效率提升很大。（21:02-21:25）
- 内测时，原本目标用户已经被触达，反应不错，但出现了一些没想到的人群。（21:52-22:10）
- 第一类意外用户是“真的要做一些视频给消费大众的人”，比如漫剧从业者。他们不需要很强的最终成片能力，只要分镜能对得上，把所有分镜下载下来自己剪就够了，因此觉得分镜效率很高。（22:10-22:45）
- 第二类是完全没做过视频、不爱出镜、想把自己的状态用动画方式展现出来的人。Nono 说动画是一个容易表达精神世界的载体，这类是“素人用户”，不是原来自媒体人。（22:45-23:12）
- 第三类不是为了给大众看，而是为了社交关系看：家长给孩子、学生给老师、情侣之间、养宠的人给宠物做、成年人给父母做等。（23:12-23:33）
- 关于后续是否做减法，Nono 说会有点类似当时抖音做垂类的思维：比如做科普类，就找科普类 UP 主，看所有科普类动画视频的结构，把它变成知识库，再服务他们。她说现在只是这些用户在用，还没有服务好，会一步一步把每个方向啃下来。（23:33-24:01）

**例子或论据原样呈现**

- 漫剧用户只要分镜能对得上，下载分镜自己剪。（22:10-22:45）
- 素人用户不爱出镜，希望用动画表达自己的状态。（22:45-23:12）
- 家长给孩子、学生给老师、情侣之间、养宠的人、成年人给父母做视频。（23:12-23:33）
- 科普类会去看科普 UP 主动画视频结构，做成知识库。（23:33-24:01）

### 24:01-27:31 Agent 工具会不会变复杂；自由度与稳定性的架构取舍

**这段在谈什么 / 主持人问了什么**

主持人提到 Maya、剪映都可能变复杂，问 OeOe 会不会也变得很复杂；又问 Agent 的灵活性与可控性如何平衡。

**嘉宾的回答（带时间戳）**

- Nono 说 Agent 是一个很好的“载体”，因为以前创作工具常从简单到臃肿，再被更简单工具替代。她举设计行业 PS、Sketch、Figma，以及剪辑行业 PR、Final Cut、剪映的例子，说这些工具核心都是功能往上堆，组织会变臃肿。（24:01-25:01）
- 她认为 Agent 也会有功能往上叠的一面，但不会太臃肿，因为很多东西藏在内部，不是用 GUI 方式展示。她说 Agent 有魅力之处在于用户和产品共建，用户有探索能力时，会用到团队都不知道的功能。（25:01-25:42）
- 她说现在 OeOe 的很多不足，是用户自己在想办法解决；Agent 不是死的东西，用户可以不断探索边界，工具也会变得丰富。（25:42-25:57）
- 关于自由度和可控性，她说主流 Agent 大致有两类：一类自由度高、以语言模型为主；一类以 Workflow 稳定性为主。OeOe 想同时要流水线稳定性，又要能对流水线上的每个 Agent 自由对话，因此架构要求非常高。（25:57-26:31）
- 她说短短两个月里团队做了四次架构迭代。第一版用 system prompt 定义各种 Agent，让模型自己判断 Workflow，但模型不听话，不按要求交给下一个 Agent，自由度太高；第二版是严格 Workflow，但没有修改自由度；第三版是在 Workflow 基础上让它有信号，知道什么时候跳出来修改，修改完再进去；第四版是在第三版基础上强化跳到其他过程的自由度。（26:31-27:20）
- 她说第三、第四版仍在调，是很容易出问题的架构；用户碰到的很多问题包括“推不下去了”“怎么又不动了”，可能是跳出来忘记进去，或者没跳出来一直往下走。（27:20-27:31）

**例子或论据原样呈现**

- PS → Sketch → Figma，PR / Final Cut → 剪映，作为创作工具复杂化与替代的例子。（24:30-25:01）
- 两个月四次架构迭代：自由 Agent、严格 Workflow、Workflow 中跳出修改再回去、在此基础上增加跳转自由度。（26:31-27:20）

### 27:31-32:58 OeOe 与剪辑工具、视频编辑创业、多个 Agent 的角色设计

**这段在谈什么 / 主持人问了什么**

### 35:31-43:31 微信体系、字节体系与产品观

**这段在谈什么 / 主持人问了什么**

主持人说 Nono 同时在微信体系和抖音 / 字节体系工作过，问两套产品体系里做产品经理的感受、相同和不同。

**嘉宾的回答（带时间戳）**

- Nono 说自己在微信不是最重要岗位，当时做 QQ 相册，是产品经理很初级阶段，但微信对她最大的影响是培养了“非常深刻的产品价值观”。她说张小龙在很多深入的人性思考上，能把思考嫁接到产品上，这是很厉害的功力。（35:31-36:31）
- 举例时，她说每更新一个版本都会做版本语言。她记得当时微信有一句“小龙说我所说的都是错的”，搭配了一个麦克蒂森的图。她说这看起来简单，但背后隐藏了很多思考，最终表达出来的广告语有力量。（36:31-37:10）
- 她说在微信体系里，每天都要看用户反馈，可能一天一百多条或一千多条，要识别哪些是真需求、哪些是假需求。她提到张小龙产品公开课里说，识别需求的方法是“泡到用户的反馈里面去”，看用户到底在说什么，而不是在会议室里争论。（37:10-38:13）

- 转到字节体系，她说字节非常注重数据，自己开始会有些不适应，因为数据好不代表体验最佳。比如把一个按钮变很大，数据会好，但不一定适合用户体验。她最初抵触纯粹数据导向的产品体系。（38:44-39:01）
- 她说自己刚加入字节时不是剪映产品一号位，而是做特效，后来做剪映，也在抖音做过一阵。剪映更有产品体验感；特效业务让她慢慢体会到数据力量。（39:01-39:30）
- 她举“投稿率”的例子解释策略产品：根据用户各种行为判断他使用某功能的概率，例如经常打开特效、经常看特效类视频、点过转盘、点过收藏等，再把风先造给大概率会使用的人。（39:30-40:00）
- 她说懂了原理后，会发现产品 sense 和数据可以结合：数据不是冰冷的东西，里面有用户真实行为，可能比反馈声音更真实；关键在于会解读和利用数据。（40:00-40:33）
- 她把微信和字节比作“一个右脑，一个左脑”。做 AI 产品时，微信层面培养的产品感觉、判断力，以及字节层面关于模型搭配和策略的理性思维，都有帮助。动画本身也是艺术和技术融合的形态。（40:33-41:02）
- 主持人问两者相同点。Nono 说相同点是“把自己的优势发挥到极致”：字节把数据科学、推荐引擎和增长体系发挥到极致；微信对用户和人性的理解非常深刻，也把这个优势发挥到极致。（41:02-42:00）
- 她还说，微信里很多决策看似由张小龙决定，但她觉得他并不是闭塞和固执的人，而是开放地听，最后在所有听的基础上收敛；字节则更相信科学决策、组织和引擎决策。（42:00-42:42）

**例子或论据原样呈现**

- 微信版本语言：“我所说的都是错的”，搭配麦克蒂森的图。（36:31-37:01）
- 每天看用户反馈，一百多条或一千多条，识别真假需求。（37:31-38:13）
- 特效投稿率：根据打开特效、看特效视频、点转盘、点收藏等行为判断概率。（39:30-40:00）
- 微信像右脑，字节像左脑；动画是艺术和技术融合。（40:33-41:02）

- 但她不认为互动性一定是很大的革新，因为要看受众是否买单。她说可编辑自由度越高，受众越专业，不一定更小白；实时性提高意味着互动性，但互动是一种主动行为，而现在用户被自媒体训练得更被动，“唯一的互动是上滑下滑”。只要多一点互动，人群就会变窄。（58:16-59:01）
- 她又说自己“不设限”。因为过去判断时会出现 Sora2 这种变化：它只是做到了切镜自然，听起来不是很大的改动，但效果非常好。也许一些判断中的巨大改革不会对消费人群带来很大改动，反而是在现有媒介基础上的小改动会带来大的受众心理变化。（59:01-59:52）
- 关于不同模型，Nono 说早期 Task Agent 会学习动画里的镜头需求。如果有打斗镜头，会调用海螺，因为海螺打斗更好；人物表情、情感细腻的镜头，可能 C 的表现更好；CG 感、特效大片感，可灵会更好。团队会根据镜头描述自动分配。（59:52-01:00:31）
- 她认为模型差异有先天和后天之分。先天是数据标准、分布不同，输出东西完全不同；后天是模型公司战略，例如想做影视级大片，就训练很多影视级素材，强化某一环节后，模型在那方面会更强。（01:00:31-01:01:31）
- 她说视频模型很难真的“大一统”。即使模型完全一样，调用手法、喂进去的东西不同，出来的东西也不一样。她举 Sora2 的例子：之前没有喂场景图，只喂小元素，场景一致性不强；后来喂的元素有一点改变，输出就完全不一样。小改动在模型后面这一层都会带来大影响，更别说模型训练数据维度不同。（01:01:31-01:02:22）
- 主持人问 OeOe 里用户能否选模型。Nono 说暂时没有，后面可能会做，这是团队现在自己的选择。（01:02:22-01:02:31）
- 关于 2026 年或更往后视频模型厂商竞争，她说大家都会加强自己擅长的部分，补足不擅长的部分；模型厂商仍会更通用；大的方向是实时性和可编辑性增强。（01:02:31-01:02:58）

**例子或论据原样呈现**

### 01:02:58-01:06:21 天使投资、自我介绍、“容器”

**这段在谈什么 / 主持人问了什么**

主持人问如果有 300 万美金做天使投资，会分三份投给谁；最后请 Nono 用“十个我是……”介绍自己。

**嘉宾的回答（带时间戳）**

- 主持人说那第一个“我是”就是“我是一个正在尽量减少我是什么什么的人”。Nono 接着说，如果一定要说一个“我是”，她觉得自己是“一个容器”。（01:04:30-01:04:49）
- 她说这仍和 OeOe 有关：感觉有一个“动画之神”通过她这个容器表达某个东西，在这个时代可能有更好的表现方式；不是她在表达，而是有个东西要表达，而她这个容器适合。OeOe 本身也不做表达，而是在帮助想表达的人更好地表达。（01:04:49-01:05:21）
- 她说这个灵感让她想到李安采访。李安说比他有才华的人很多，他们没有机会做成这样的成就，不是没有才华，而是“电影之神选中了我”，他是一个容器，把电影这件事通过他表达出来。Nono 说自己某种情况下也有这种感受，希望自己变成一个“通道”，让大家去表达。（01:05:21-01:05:59）
- 结尾，主持人希望过一段时间再聊一次，那时 OeOe 可能有更多进展，也看到更多人通过 OeOe 这个容器做出更多产品。Nono 回应“希望”。（01:05:59-01:06:21）
