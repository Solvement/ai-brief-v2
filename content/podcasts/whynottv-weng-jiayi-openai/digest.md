### 02:33-08:53 童年：数学、学习慢与提前投资未来

**这段在谈什么 / 主持人问了什么**

主持人问翁家翌小时候是什么样的小孩，之后追问他为什么从小喜欢数学、为什么初中就提前学高中数学，以及这种兴趣是否来自家庭或外界。

**嘉宾的回答 / 原话转述**

翁家翌说，他小时候“特别喜欢学奥数”，从一年级开始一直学到六年级，初中也继续学（02:41-03:06）。他发现自己做数学题比别人快，比如二年级口算题别人还没做完，他已经做完；他说这种题对他来说“不用过脑子”，像现在说的 System 1，“直接看一眼出来答案”（03:08-03:30）。

但他同时说自己“学习还算相比于其他人算比较偏慢”，学一个新东西经常要花别人 2 到 3 倍时间，现在读代码也是这样，要花很多时间理解整个 context（03:36-04:02）。他解释说，自己可能需要更长时间构建“知识树”，一旦理解之后就用得很快；别人可能从根到顶层一步步推，他则会建立 shortcut，直接连到结论（05:15-05:43）。

主持人问他是不是从小意识到自己更聪明，他连续否认：“没有，没有这种感觉”（05:46-05:51）。被问为什么愿意提前学数学，他说：“我想投资未来，我想投资我自己的未来”（06:02-06:07）。他的逻辑是，与其把时间浪费在刷题上，不如学一点对未来有用的东西，后面的收益可能更多（06:18-06:27）。

谈到兴趣来源时，他说首先是感兴趣，觉得初等数学太无趣，稍微有意思一点的数学更能引起兴趣（06:51-07:08）。他接受主持人说法：小学时的成就感形成初始 reward，后来每次数学技能树点上去都有正反馈（07:19-07:35）。相比之下，他说自己打篮球经常被舍友按着打，跆拳道一打实战也被对手揍，所以没有正反馈（07:37-07:56）。

谈到编程，他说不是高中才开始，而是初一开始；小升初去了时代中学，学校有编程兴趣班，父母觉得可能有意思，他就试了试，觉得还可以（08:15-08:53）。

**例子或论据**

- 背课文例子：睡前想尽办法磕磕巴巴背出来，睡一觉第二天醒来就倒背如流（04:11-04:36）。
- 提前学习例子：初中开始上高中数学，初二把高中内容啃完，初三开始学微积分（04:43-05:04）。
- 正反馈例子：数学给他成就感；篮球、跆拳道没有给他正反馈（07:19-07:56）。

### 08:53-16:03 高中信息学竞赛、NOI 与代码优化

**这段在谈什么 / 主持人问了什么**

主持人转向高中，问他为什么搞 OI 信息学竞赛、省选和 NOI 经历如何、竞赛失利对升学有什么影响，以及高三是否完全停止编程。

**嘉宾的回答 / 原话转述**

翁家翌说搞竞赛很大程度上是因为升学压力：对非北京学生来说，不搞竞赛想上清华北大“难如登天”（09:02-09:18）。他高一花了一点时间做数学竞赛，但学校没有往上走的基础，自己学也困难，且觉得省选往上的数学题非常困难，于是放弃数学竞赛，转向 OI（09:21-10:10）。

他介绍信息学竞赛流程：先 NOIP，提高组省内选拔；各省按不同标准选省队参加国赛；福建省选有两轮，一轮寒假，一轮 4 月；五六月有清华或北大夏令营，七八月是 NOI 国赛（10:17-11:03）。

他回忆自己的过程：高一省选没过，“完全不会做”；高二会做一点，会一些 heuristic 方法，比如“最小双圆覆盖”一题搞出 70 分，刚开始好像全场最高，因此进了省队（11:08-11:42）。清华夏令营拿到“无条件降 60 分、有条件本一线”的录取优惠（11:49-12:30）。但 NOI 国赛没有拿到有条件本一线，他说这是“黑历史”：自己是福建省队倒数第一，福建当年只有一个铜牌，就是他，其他人都银牌以上（12:30-12:50）。

这次失利让他对升学产生不确定感，因为高二下半年没有搞文化课，不知道高考能考多少分；他害怕万一 60 分加分全用完仍不够（12:50-13:25）。但最后在家人鼓励下选择清华降 60 分，而不是更保险的上海交大本一线录取（13:35-14:07）。

高三他并没有完全不碰代码，而是会“偷偷地碰”（14:09-14:18）。他尤其对代码优化有追求，甚至练到在 iPad 的 Safari 上没有编译器的情况下，用 iPad 键盘裸打代码直接提交；他说这锻炼了他对整个题目和逻辑的完整认知、快速定位错误的能力（14:21-15:08）。他还喜欢 OI 里的常数优化：同样 N² 算法，比较实际运行时间；如果运行一样快，再比代码长度。他会同时优化速度和代码长度，觉得“虽然说没什么用，但是很有意思”（15:15-15:53）。

**例子或论据**

- “最小双圆覆盖”题拿 70 分、刚开始像全场最高（11:32-11:40）。
- 清华夏令营优惠：无条件降 60、有条件本一线；银牌线/前 150 名可本一线，金牌可直接保送（11:49-12:30）。
- iPad 裸打代码提交、没有编译器（14:21-14:42）。
- OI 常数优化：按测试点运行时间排序，若一样快按代码长度排序（15:15-15:49）。

### 16:03-19:24 清华本科：开源作业与打破信息差

**这段在谈什么 / 主持人问了什么**

主持人问他 2016 年开始读清华本科后，大一大二最值得回忆的事是什么。翁家翌第一反应是把所有作业都开源。

**嘉宾的回答 / 原话转述**

翁家翌说，真正浮上脑海的事是“我把我所有的作业都开源了”（16:16-16:24）。他把收集到的“上古作业、上古材料”放到 GitHub 上，除了有版权问题的没开源，其他没有版权问题的都开源了（16:34-16:48）。

主持人问为什么，他回答：“因为我觉得应该打破信息差”（16:52-16:56）。他认为信息差在清华生存中很有用，但每个人都应该平等拥有这些信息（16:56-17:05）。他开玩笑说，现在回去随便抓一个计算机系学弟，可能不认识捐新系楼的人，但应该认识翁家翌，因为“大家都看我的作业活的”，比捐楼有用（17:07-17:35）。

他进一步解释，自己不想让后人重蹈覆辙、疲于奔命。很多人不是不擅长学习，而是不擅长搜集东西；如果给他们信息平权的机会，他们在清华会活得更好（17:46-18:12）。信息平权包括往年作业、学长学姐问过的重点、老师没有规定不能放出的作业（18:12-18:26）。否则学生可能花十几二十个小时钻牛角尖，又不敢问助教，收益很低（18:26-18:52）。他希望开源作业能帮后来的学弟学妹拥有更多自己的时间，去做他们更想做的事，而不是做这些“比较浪费生命的事”（18:52-19:12）。

**例子或论据**

- 清华作业 GitHub repo 在计算机系广为流传（19:14-19:20）。
- 学生因不知道怎么做作业而花十几二十个小时钻牛角尖（18:26-18:52）。

### 19:24-28:07 入门科研：朱军组、RL、图形学与 VizDoom

**这段在谈什么 / 主持人问了什么**

主持人问他大一大二大三何时入门科研。对话随后谈到他如何进入朱军老师组、为什么选强化学习、他对 AI/图形学/网络安全的兴趣，以及第一个强化学习项目 VizDoom。

**嘉宾的回答 / 原话转述**

翁家翌说入门科研应该是大二。他问学长推荐实验室，学长给了朱军、唐杰、崔鹏三个名字；他以为顺序代表“牛逼程度排序”，于是选了朱军老师（19:31-20:10）。见面后朱军老师给了三个方向：贝叶斯、GAN、强化学习。他当时想搞 GAN，因为觉得图像生成有意思，但“不知道哪个是 GAN”，误打误撞选了 RL；他以为 RL 是搞图像的东西（20:14-20:52）。后来发现 RL 是“打游戏的东西”，觉得挺有意思，就一直做下去（20:54-21:03）。

他说自己最开始有三个大的兴趣方向：AI、图形学、网络安全（21:11-21:25）。网络安全让他觉得 hacker 很酷，他大学业余做过不少相关东西，也给学校修过校园网 bug（21:24-21:36）。图形学对他来说也是一种 hacking，是“对现实世界的 hacking”，可以用自己的视角构建脑中想象的场景，让人觉得自己像上帝、像神（22:01-22:23）。他小时候看过电影 TRON（《创：战纪》），被特效震撼，希望有一天能做出那样的特效或构建自己的虚拟世界；图形学是实现这个的手段（22:27-23:08）。

大二上图形学时，他因为喜欢花很多时间做大作业，拿到全班唯一或唯二的 A+；还发明了一个减少迭代收敛次数的新算法，用大量算力渲染了一张 16K、没有噪点的图（23:11-23:55）。但他没有继续图形学，因为觉得搞科研还是专心比较好，不能脚踩两条船；当时已经在朱军老师组做 RL（23:59-24:17）。

第一个 RL 项目是 VizDoom：用神经网络在固定地图里让 Doom 游戏通关，包含杀怪、捡血包、避障、到达终点等任务；这个项目拿了冠军（24:17-24:56）。但他“不享受”这个科研过程，因为 task 环境太单一，需要疯狂 overfit，用各种技巧防止训练崩；即便训练没崩，也不知道怎么调参才对。他说这是真的炼丹，调参难度比 CV 难 10 倍、100 倍，都是玄学，要靠 heuristic 避免 corner case（24:56-25:40）。

他认为在当时的任务上，改算法没有那么本质，大部分情况下不能 work；对于人类简单的任务，对 RL 来说可能非常难，因为 RL 需要大量知识，比如什么是障碍物，这对人不需要反应，对 AI 是不同难度（25:40-26:41）。因此他有意识地把重心转向“如何帮助这一类科研更顺利进展”，想在大四造一套 RL infra 或小 library，让想在这个领域卷的人更方便地卷（26:57-27:30）。他觉得自己擅长软件工程、重构代码、做好用户体验；但对“这个东西怎么调”有生理上的排斥（27:30-27:55）。

**例子或论据**

- 给学校修成绩单下载 bug：成绩单一次 10 元，他和学长发现可以一分钱或不要钱下载，下载几次后把 bug 反馈给教务部门修掉（21:36-22:03）。
- TRON/《创：战纪》带来的图形学兴趣（22:27-23:08）。
- 图形学大作业：A+、新算法、16K 无噪点渲染图（23:11-23:55）。
- VizDoom 任务：出生点、终点、杀怪、捡血包、避障、通关（24:17-24:50）。

### 28:07-40:02 暑研、MoE/NLP、申请挫败与 GPA 评价体系

**这段在谈什么 / 主持人问了什么**

主持人问大三申请暑研的节奏，以及暑研后申请 PhD/Master 的状态。随后对话转到 GPA、PhD/Master 鄙视链和自建评价体系。

**嘉宾的回答 / 原话转述**

翁家翌说，申请暑研是为了大四申请，发了一堆套磁信但没什么结果，后来导师帮他联系到 Yoshua Bengio，于是去了 Mila（28:07-28:34）。他回忆 2019 年暑假，Yoshua 刚拿图灵奖不久，给他和带他的 postdoc 的任务是做一个类似 MoE 的东西；当时没有 MoE 的成熟概念，他们只知道有一个 router 可以选择不同 path，让他去实现（28:41-29:28）。

这个任务和 RL 没关系，是纯 language model / Transformer。他说自己是做 RL 的，却被招去做 NLP，觉得很奇怪；由于不擅长快速学新东西，他花了很长时间入门 Transformer/NLP，搓出一个东西但效果不好（29:31-29:51）。从现在看，他觉得即便方向对，也需要算力、工程能力和 scale up，当时一个人几块卡，做不出来（29:51-30:12）。主持人说他先搞 RL、又被 Yoshua 让去搞 NLP，像是被命运推向 OpenAI 技术路线；他回应“强行这么说也可以”，并说“马后炮是没有用的”（30:12-30:35）。

谈到当时对 NLP 的看法，他说 NLP task 太分散；当时很多人想把 RL 用在 language model training 上，但问题还没有解。Transformer 直接跑强化学习会崩，大家还没想出怎么防止崩（30:35-31:13）。后来他认为环境要是纯文本，reward 要强；而之前有人尝试把 Transformer 用在 Atari decision、图像 embedding 或结构化游戏信息上，且 Transformer 随机初始化，难度不同（31:13-31:55）。

他承认自己当时 context 不够、对领域认识不够，所以非常挣扎；主持人问是否处在“怎么都不可能把东西搞 work”的状态，他回答“是的是的”，原因是算力不够、对 MoE 认知不够、工程能力和 compute 都不够（31:55-32:49）。暑研结束后，他状态不太好，因为周围同学去 Stanford、CMU 暑研搞出一作 paper，而他只拿了一封推荐信，不知道信里写得怎样，也没有底（32:49-33:36）。

申请季他是按 PhD 申请，但只拿到 Master（33:36-33:44）。主持人说自己当年在知乎上看到他的申请结果，觉得竞争激烈。翁家翌说，当时有一点挫败，身处清华小环境，很难摆脱“PhD 比 Master 好”的氛围和鄙视链；但后来觉得很多时候学历高低并不成立，取决于你到底干了什么（34:16-35:13）。

他随后说，自己一直想做让自己与众不同的事情，尝试挣脱固有评价体系（35:17-35:37）。他认为 GPA 不是唯一评价体系，需要在大学里找到适合自己的评价体系，为它奋斗，让自己开心（35:38-35:55）。从需求方看，如果目标是找工作，招人者更看重相关经验而不是 GPA；匹配当前工作的经验可以抵好几年工作经历，哪怕是 new grad 也没问题，因为招人最主要是“招进来能用、能干活”（36:16-36:50）。

他本科时意识到应该创造自己的评价体系，而不是用别人提供的评价体系；他采用了导师苏航推荐的非官方评价体系：计算机系有三个指标——论文、比赛、GitHub star 三位数以上（37:13-37:45）。这让他意识到可以在开源社区做一些事情，让自己有与众不同的地方，而不是花很多时间刷 GPA（37:45-38:13）。

主持人问如何处理本科三四年内重要、三四年后又不重要的 GPA 矛盾。他说自己会“最低限度投入”，达到想要的要求即可，再往上一点都不愿意投入，“多一分都不想花时间”（38:44-39:11）。比如期末前算一下已有多少分，安排考试要不要复习；如果 87 分是 B+，他就很满足（39:11-39:43）。

**例子或论据**

- Mila 暑研：类似 MoE 的 router/path 任务，2019 年当时没有成熟 MoE（28:41-29:28）。
- 做不出来的原因：几块卡、工程能力不够、无法 scale up（29:51-30:12；31:55-32:49）。
- 清华评价体系例子：论文、比赛、GitHub star 三位数以上（37:31-37:43）。
- GPA 投入方式：最低限度，够用即可，87 分 B+ 就满足（38:44-39:43）。

### 40:00-48:09 疫情、天授 Tianshou：从需求出发写 RL 框架

**这段在谈什么 / 主持人问了什么**

主持人接着问疫情时期出国的不确定性，以及那段时间做天授和 tuixue.online 的项目。先聊天授：为什么做、是否为了论文、做起来多难、为什么受欢迎、如何避免开源项目腐化。

**嘉宾的回答 / 原话转述**

主持人问有没有功利考虑，比如发一篇 NeurIPS paper。他说：“没有，我不想发 paper，我觉得发 paper 完全没有意义”（42:36-42:49）。原因是他已经有 paper，多一篇少一篇没意义，申请也够用了；比赛也有了；GitHub 三位数 star 可能课程 repo 算有，但他想有一个“正儿八经”的三位数（42:50-43:15）。

**例子或论据**

### 01:02:00-01:09:32 PhD、工业界与工程能力 / infra 的价值

他曾给职业生涯定过一个指标：最大化自己名字在 OpenAI blog 上出现的次数（01:08:45-01:09:06）。为实现这个 reward，应该先做 infra；单个 research 不能 scale up，而 infra 大家都用，可以 scale up；他又擅长 RL infra，所以这是非常适合的机会（01:09:06-01:09:27）。

**例子或论据**

### 01:13:08-01:19:57 OpenAI 初期、ChatGPT、RLHF 与初印象

主持人进入 OpenAI 主题，先问强化学习和 post-training 是什么，再问 2022 年 7 月入职时 ChatGPT 是否已是主线、内部是否预见到成功，以及 OpenAI 初印象。

主持人问当时是否能想象 ChatGPT 大规模成功。翁家翌说“没有”（01:16:00-01:16:17）。他内测时能看出很多不足，第一次用觉得只是一个会说话的模型，“那也就这样吧”；用了几次发现能帮他解决一点代码问题，但有限，也没有那么想用（01:16:17-01:16:43）。他认为自己提前知道这个东西，有循序渐进过程，不觉得突然；但发布后展示给身边人看，他们觉得很突然，这是他没想到的（01:16:43-01:17:06）。

他说当时发 ChatGPT 也只是为了看看能否收集真实世界用户数据，可能收完 5 天就关，如果没人用的话；他们期待可能一开始一万两万用户，然后跌回去，但实际 curve 是指数（01:18:46-01:19:17）。

谈到初入 OpenAI 的体验，他感觉那是一个“大号实验室”，没有想象中那么有方法论，但里面有很多很强 research 直觉的人，可以指明方向，然后去做。Barret、Luke、Liam 从 Google 来并加入 John Schulman 的 RL team 后，他们组开始引进“谷歌的先进生产力”（01:19:19-01:20:03）。
