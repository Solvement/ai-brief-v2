# Paper · 论文本身

## 一句话总结

想把一个**会长篇推理的大模型**(老师)的本事,搬进一个**更小、更便宜的小模型**(学生)里,最省事的路线是 **On-Policy Distillation(OPD,在策略蒸馏)**——让学生自己写推理轨迹,老师在学生自己写出的 token 上给监督。但作者指出:师生分布差距一大,学生会写出老师几乎不给概率的 token,这时常用的 **K1 reverse-KL 估计器**会给出极端负的 policy-gradient 信号,训练可能被少数离群 token 带崩。TrOPD 的做法是**逐 token 分区治理**:老师信得过的 token 进 trust region,继续用 reverse KL 学;老师信不过的 outlier token 改用 teacher top-k forward KL 给温和监督;再用老师前缀 + 学生续写的 off-policy guidance 帮学生从高质量区域开始探索。论文自报在数学、代码、指令遵循、STEM 上相对 OPD 分别提升 **+3.34 / +4.00 / +5.11 / +6.18** 点。[^arxiv]

## 问题(Problem)

- Large Reasoning Models(LRMs,大推理模型)靠更长的 test-time reasoning 提高数学、代码、agent 等任务表现,但推理成本高;Small Reasoning Models(SRMs,小推理模型)要靠蒸馏把能力压下来。[^arxiv]
- **off-policy distillation(离策略蒸馏)** 是让学生抄老师生成好的答案;问题是训练时看到的是老师轨迹,推理时走的是学生自己的轨迹,这叫 **exposure bias(暴露偏差)**,长链推理里尤其明显。[^opd]
- **OPD(在策略蒸馏)** 改成让学生自己生成 Student-of-Generation(SoG),老师在学生轨迹上给分,更接近推理时真实分布。
- 但 OPD 在长推理上不能便宜地算 full-vocabulary KL。全词表 KL 的显存是 `O(n·k)`(n=序列长,k=词表大小),所以现有推理 OPD 常用 token 级 **K1 estimator(K1 估计器,只看采样 token 的省显存估计)**。这带来两个瓶颈:[^bottleneck]
  1. **policy-gradient outliers(策略梯度离群点)**:当老师给学生某个 token 的概率接近 0,梯度形如 `(1 / π_S(x)) · log(π_T(x) / π_S(x)) -> -∞`,少数 token 就可能主导训练。
  2. **low-quality SoG(低质量学生生成)**:难题上学生自己写不出好开头,OPD 只能在低质轨迹上学,有效监督空间被锁死。
- 前人 REOPOLD 用 reward clipping,entropy-aware OPD 用高熵 token 筛选;本文的基准结果显示,这些方法不是不稳定,就是把有用监督一起裁掉。[^bench]

> [!key] 立场
> 这篇真正值得学的不是某个新 loss 名字,而是一个工程判断:**强老师给弱学生的逐步监督不是天然可靠的;要先判断"这一步老师是否信得过",再决定用哪种监督信号。** 这比固定阈值裁剪更像生产训练系统:可靠区照常学,不可靠区降级成温和回退,冷启动再用老师前缀扶一把。

## 关键术语(Key terms)

| 术语 | 大白话解释 |
| --- | --- |
| **distillation(蒸馏)** | 用大模型(老师)的行为/概率分布训练小模型(学生),让小模型低成本复用老师能力。 |
| **on-policy / off-policy(在策略 / 离策略)** | on-policy=学生用自己生成的轨迹来学;off-policy=学生照老师已生成的轨迹学。本文主攻 on-policy。[^opd] |
| **reverse KL / forward KL(反向 / 正向 KL)** | KL 是衡量两个概率分布差距的量。reverse KL 通常偏向抓老师最自信的模式(mode-seeking);forward KL 更像覆盖老师分布(mode-covering)。 |
| **K1 estimator(K1 估计器)** | 不在整个词表上算 KL,只用学生采样到的 token 估计,把 long-CoT 蒸馏的显存从全词表开销降下来;代价是遇到师生分布错位时更容易出现离群梯度。[^bottleneck] |
| **trust region(信任区)** | 对某个学生 token,如果老师也给出相对高概率,说明老师能提供可靠监督,这个 token 进入正常 OPD 区。 |
| **outlier(离群区)** | 学生写出的 token 在老师分布下很不自然;这时继续用 K1 reverse KL 可能给出极端梯度,需要换监督方式。 |
| **speculative decoding acceptance ratio(投机解码接受率)** | 投机解码里老师用 `min(π_T/π_S,1)` 判断是否接受学生草稿 token。本文借这个比值当"老师认不认这个 token"的信任度。[^trust] |

## 核心方法(Core method)

TrOPD 把一条学生生成轨迹拆成三类区域,每类用不同目标:

1. **On-policy trust region**:学生自己生成 `x ~ π_S`;若 token 以 `P_trust(x)=min(π_T(x)/π_S(x),1)` 采样进入信任区,就用 K1 reverse KL 估计器学习。直觉是:老师给这个 token 的概率不比学生低太多,老师的监督比较可信。[^trust]
2. **On-policy outlier**:学生仍然生成 `x ~ π_S`,但 token 落在离群区时,不用那个会爆的 reverse-KL 梯度,改用老师 top-k 词表上的 forward KL。大白话:学生刚写了老师不认可的词,这时别围着这个坏 token 惩罚;改看老师最可能写的 top-k 词,给学生一个更温和的正面方向。[^outlier]
3. **Off-policy guidance**:老师先写一段前缀 `x[:l]`,学生从 `x[l:]` 继续写,前缀用 forward KL 模仿。训练初期最大前缀长度等于最大训练序列长,随后用 cosine schedule 退火到 0,最后回到完全 on-policy。系数 `β=0.001`。[^guide]

> [!key] 补丁:为什么不是"直接丢掉离群点"
> 论文先试了 Mask Outlier 和 Clip Outlier。Mask/Clip 确实比 vanilla OPD 稳,并且 Mask Outlier 在图中维持了更高 policy entropy、较低 gradient norm;但只屏蔽会丢掉离群区里仍然有用的老师信号。TrOPD 的 FKL Outlier 试图保住这部分信号:离群 token 本身不可靠,但老师 top-k 分布仍可当正面示范。[^outlier]

> [!warn] 小心一个公式直觉坑
> 原文在 outlier FKL 解释里说,当学生在老师 top-k 上的概率和趋近 0 时,辅助 outlier objective 会被抑制、不干扰 trust region。这个说法按常规 KL 直觉并不显然;复现时应以作者代码/实现为准。但当前 GitHub 仓库尚未发布训练代码,所以这里只能按论文表述记录,不替它补实现细节。[^repo]

## 架构 / 流程(Architecture / pipeline)

```mermaid
flowchart TD
  P([训练 prompt]) --> S[学生生成 SoG<br/>x ~ pi_S]
  S --> A{逐 token 算信任度<br/>P_trust=min(pi_T/pi_S,1)}
  A -->|M_x: trust region| RKL[On-policy trust region<br/>K1 reverse KL<br/>log pi_T/pi_S<br/>Memory O(n)]
  A -->|not M_x: outlier| FKL[On-policy outlier<br/>teacher top-k forward KL<br/>Memory O(nk)]
  P --> T[老师生成前缀 x[:l]]
  T --> C[学生续写 x[l:]<br/>off-policy guidance]
  C --> G[beta * forward KL<br/>beta=0.001<br/>前缀长度 cosine 退火到 0]
  RKL --> L[TrOPD unified objective]
  FKL --> L
  G --> L
  L --> U[更新学生模型]
```

## 创新点(Innovation points)

| 创新 | 新在哪 | 为什么重要 |
| --- | --- | --- |
| 统一 OPD 基准 | 把 OPD、EOPD、REOPOLD、entropy token selection、FKL/JSD 等放到同一训练设置下比较 | 先定位真问题:不是"缺一个更强老师",而是 K1 reverse KL 在分布错位时监督不可靠 |
| 自适应信任区 | 用 `min(π_T/π_S,1)` 而不是手设阈值 `τ` 划 reliable token | 少一个固定阈值超参,让"老师是否接受学生 token"直接由师生概率比决定 |
| 离群区 top-k FKL | outlier token 不再用爆炸 reverse-KL 信号,改从老师 top-k 分布给正面示范 | 比纯 mask/clip 多保留可用监督,消融里最高 |
| 老师前缀 + 退火 | 初期老师给高质量开头,学生续写;训练后期前缀长度退火到 0 | 缓解低质量 SoG,但最终仍回到学生自己的 on-policy 分布 |
| 与 AOPD 可叠加 | TrOPD + AOPD 从 40.63 提到 41.67 | 说明 trust-region 分区和 AOPD 的 token 级利用/模仿是两条可组合轴 |

## 限制与风险(Limitations and risks)

- **作者自认缺真实部署/应用研究**:Limitations 明说没有 practical deployment 和 application studies。[^limits]
- **只覆盖 OPD post-training**:论文聚焦 DeepSeek-Qwen2.5-1.5B 与 Qwen3-SFT-1.7B 的 OPD 后训练;作者认为真实高性能 SRM 可能还需要 pre-training / mid-training,否则性能上限受限。[^limits]
- **依赖老师分布可信**:trust region 的尺子是 `π_T/π_S`;如果老师本身不可靠、或者师生 tokenizer/能力差异导致概率比不可比,这把尺子可能失真。原文未充分展开这类失败模式。
- **超参敏感性披露不足**:`k=64`、`β=0.001`、cosine 退火日程都给了,但跨任务敏感性没有系统表。
- **代码未释放**:当前仓库没有训练/评测实现,所以很多实现细节(例如 outlier FKL 的数值稳定处理)只能以原文表述为准。

## 技术细节(选读)

> **三区目标按原文 Table 2**
> On-policy trust region:`x ~ π_S`,`-KL(π_S || π_T)`,估计器 `log(π_T(x)/π_S(x))`,显存 `O(n)`;on-policy outlier:`x ~ π_S`,`-KL(π_T || π_S)`,估计器为老师 top-k 上的 `Σ π_T(v) log(π_S(v)/π_T(v))`,显存 `O(nk)`;off-policy guidance:`x ~ π_T`,`-β KL(π_T || π_S)`,估计器 `β log(π_S(x)/π_T(x))`,显存 `O(n)`。[^table2]

> **Qwen3-Nemotron-4B 老师怎么来的**
> Appendix A 写明,Qwen3-Nemotron-4B 从 Qwen3-4B-Base 初始化,先用 Nemotron 3 Nano 公开 SFT 数据/recipe 做 SFT,组合多域数据后约 **14M training samples**;再用 Nemotron-3-Nano-RL-Training-Blend 做 RLVR,四域样本量为 Math **22,056**、Coding **19,169**、Science **19,670**、Instruction Following **16,575**。SFT 用 Adam,lr **5×10^-5**,weight decay **0.1**,warmup **10%**,batch size **512**,平均 response length 约 **7K tokens**;RLVR 用 GRPO,group size **16**,batch size **128**,每 **2048 rollouts** 更新,最大生成 **32K tokens**,temperature **1.0**。[^teacher]

## 后续演化 · 这方法后来怎样了

TrOPD 是 2026-06-03 v2 的新论文,截至本轮深读没有明确后继工作可可靠归因。下面是论文自身引用和同赛道脉络,不是"后来优化了它"。

- **REOPOLD**(arXiv:2603.11137,*Scaling reasoning efficiently via relaxed on-policy distillation*)——本文最强基线之一,用 reward clipping 处理 outlier;TrOPD 认为它可能同时裁掉 informative supervision _[置信度:高]_。
- **AOPD**(arXiv:2605.06387,*Asymmetric on-policy distillation*)——并发工作;本文 Table 6 显示 TrOPD + AOPD 可从 40.63 到 41.67 _[置信度:高]_。[^aopd]
- **EOPD / Entropy-aware OPD**(arXiv:2603.07079)——用熵选 token;本文在统一基准里显示 entropy selection 不稳定,在某些设置下不如 OPD/REOPOLD/TrOPD _[置信度:高]_。
- **MiniLLM / GKD / Thinking Machines OPD blog**——K1 estimator、on-policy 蒸馏和 full-vocabulary distillation 的背景来源 _[置信度:高]_。
