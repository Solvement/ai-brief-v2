# Career & Learning · 求职与成长

> 假设你已读完 Paper 栏。这里只谈它对**应用型 AI 工程师 / FDE / AI-app 开发者**的你有什么用。

## 为什么这对我重要

> [!key] 它是你北极星（L3 研究 agent）的"考卷"和"现实校准器"
> 你的 north star 终点是 L3 研究 agent。RCBench 直接告诉你**这条路现在走到哪了**：最强系统 21.5/100，连复现线一半都没到，且**瓶颈是科学严谨（协议/证据/内核）而非工程**。这盆冷水很值钱——它让你不会去做"又一个会写漂亮报告的科研 agent"（专业性早就 >70 了），而是把力气押在**证据链完整性、协议保真、机制不丢**这些真瓶颈上。

更直接的用处：RCBench 的**评测设计本身**就是一套你能复用的方法论。你做 AI-Brief 的"深读冷审门"、做任何"开放式产出怎么自动判质量"的功能时，**"隐藏标准答案 + 拆加权 rubric + 钉一条复现线"**这套就是现成的范式。

## 系统设计心法

- **测能力要把"脚手架红利"和"模型本事"分开**：RCBench 用统一壳跑裸 LLM，正是为此。你评自己 agent 时也别让"我加的工具"掩盖"模型其实不行"。
- **复现线 > 满分线**：把"达到已知正确答案"定成一个明确锚点（50），比"满分 100 谁也够不到"更能测出真实差距、又给超越留空间。
- **报告质量 ≠ 内容质量**：专业性/排版是表面分，证据链完整性才是硬核——评质量时**权重要压在可验证的科学/事实子项上**。
- **多花算力不等于更对**：当任务在能力边界之外时，加预算是浪费；先确认"在边界内"再谈 scale。
- **隐藏答案防泄漏**是开放式评测的命根子。

## 作品集角度(Portfolio angle)

这能撑起一个**"开放式产出的自动质量评测"**方向的作品：给一类开放任务（科研报告、深度解读、代码方案）造一套**隐藏目标 + 加权 rubric + 复现线**的评测器，配上**错误归因分布图**。这比"我跑了个 agent"更体现评测/科学思维——而且**给 RCBench 这类 156★ 的开源 benchmark 贡献一个新领域任务或一条 rubric，是很硬的简历点**。

## 简历 / 面试故事

诚实写法：

- *"Built an open-ended-output evaluator inspired by ResearchClawBench: hidden reference, expert-style weighted rubrics with technical-keyword checks, and a 're-discovery line' anchor — to grade research-report-style artifacts beyond surface polish."*
- *"Implemented a lightweight ReAct harness (web/file/exec tools + automatic context compaction at 128k tokens) to evaluate native LLMs without scaffolding advantage."*

> [!key] 面试可讲
> "开放式产出（科研报告 / 长文解读）怎么自动、公平地判质量？" —— 用本文论证：**隐藏标准答案防泄漏、把产出拆成围绕真目标的加权可验证子项、用复现线当锚点、并把 judge 偏置用'拆细+关键词核对'缓解**。再加一句行业判断：**当前自主科研 agent 的瓶颈是科学严谨（协议/证据/内核），不是会不会写报告**——这种"既懂评测又懂现状"的回答很难造假。
