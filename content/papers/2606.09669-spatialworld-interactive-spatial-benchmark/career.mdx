## 为什么这对我重要

如果你做的是 agent 产品，迟早会遇到一个问题：demo 里 agent 能点按钮、走流程、调用工具，但一到真实任务就开始迷路、乱试、提前说完成、或者用很多无效步骤碰巧成功。SpatialWorld 给你的不是一个可直接部署的机器人框架，而是一套**评 agent 是否真的能在部分可观测世界里闭环做事**的工程范式。

> [!key] 和我们正在做的东西直接同源
> AI-Brief 的自进化/冷审门也需要这种思想：统一输入输出契约，生成器不能偷看答案；用 verifier 查最终产物，而不是看生成路径像不像；质量分和代价分分开报；再按任务类型切片找能力断崖。SpatialWorld 是 embodied spatial agent 的版本，但它的评测骨架可以迁移到我们自己的 research agent、KG 判边、深读冷审和自动回归。

## 系统设计心法

一句话：**可归因的 agent 评测，要先把接口收窄，再把判分做硬。**

- **统一接口 > 单点 leaderboard**：后端不同、动作不同、状态暴露不同，分数就不可比。先建 I/O 瓶颈，再谈模型能力。
- **验终态 > 验轨迹**：开放式 agent 会有多条合理路径；评测应看 artifact / state 是否满足条件，而不是像不像 golden trace。
- **质量线和代价线分开**：成功率高但步数爆炸，和成功率低但偶尔高效，是两种不同问题。
- **抽象对照组很值钱**：写实任务容易混入常识捷径；把语义剥掉的 3D games 类似我们给 KG 判边做“概念名替换成代号”的测试。
- **断崖比排名更有用**：Navigation-Interaction 只有 4.2% mean TSR，比“谁第一”更能告诉你下一步该修哪里。

## 作品集角度(Portfolio angle)

这篇能撑起一个很硬的作品集方向：**“agent 评测基础设施”**。不要再做一个只能展示成功 demo 的 agent；做一个能证明 agent 何时失败、为什么失败、成功代价多少、是否跨环境泛化的评测器。

你可以把它讲成三层：

1. **统一动作协议**：让多个工具/环境后端共享一套高层动作。
2. **终态 verifier**：每个任务都有可执行判分器，不靠人工看日志。
3. **能力矩阵报告**：成功率、效率、失败类型、任务切片同时输出。

## 简历 / 面试故事

诚实写法：

- *"Built an agent evaluation harness with a unified action interface, terminal-state verifiers, and dual metrics for success rate and step efficiency across heterogeneous tool environments."*
- *"Designed benchmark tasks as three artifacts: validated initial state, reference trajectory for budgeting, and deterministic terminal-state verifier; reported capability breakdowns by task complexity rather than a single leaderboard score."*
- *"Added abstraction probes that remove semantic shortcuts, revealing whether an agent relies on surface cues or robust state tracking."*

> [!key] 面试可讲
> “你怎么判断一个 agent 不是靠乱试成功？”——用 SpatialWorld 的答案：不要只报 success rate；统一接口后，用终态 verifier 判成败，再用 SE / steps / token 成本看成功代价，并按任务复杂度切片。这样才能区分真能力、接口适配和试错成本。

## 学习清单

**必学**

- POMDP / partial observability 的工程含义：只能看局部、要主动取证。
- Terminal-state verification：判最终状态，不判路径相似。
- Unified action interface：高层动作协议 + 后端 adapter。
- TSR vs SE：质量与效率分开报告。

**有用**

- Benchmark task 三件套：初始状态、参考轨迹、verifier。
- 失败模式分类：Spatial Disorientation、Object Hallucination、Premature Termination、Action Loop。
- 抽象 probe：剥离语义捷径，单测几何/状态追踪/结构推理。

**可选**

- AI2-THOR / ProcTHOR / CARLA / VirtualHome / EmbodiedCity 的 simulator 生态。
- 多 agent collaboration eval：通信、角色分配、共享进度追踪。
- 评测成本工程：抽样、分层回归、GPU/API 成本预算。
