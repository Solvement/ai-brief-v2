## 为什么这对我重要

这是你要造的"**越用越懂你的学习 agent**"的**设计地图**。它把"怎么让一个 agent 自我进化"拆成 What/When/How/Where 四个可操作的选择题——你做任何"会自我改进"的功能时,照着它选部件、选时机、选方法,就不会从零摸索。对正在搭自进化系统(就像我们这个站的 L1)的你,这是**目录 + 决策树**。

## 系统设计心法

一句话:**"让系统自己变好"不是一个开关,是 What/When/How/Where 四个要你想清楚的设计决策。**
- **先选最便宜的进化**:改上下文/记忆 < 改提示 < 改工具/架构 < 改模型参数。能用记忆解决就别去微调。
- **任务间 > 任务内**:真正的复利来自跨任务积累(inter-test-time),不是单次任务内调一调。
- **进化必须配护栏**:自改进最大的风险是价值漂移 + 灾难性遗忘——每次自更新都要过"学新不忘旧"的回归门。
- **评估是闭环的一半**:没有能衡量"是否真变好"的评估,自进化就是盲走。

## 作品集角度(Portfolio angle)

"自进化/持续学习 agent"是最前沿、最稀缺的方向。两条路:① 按四轴框架,给一个 agent 加一个**真能跨任务变好**的自进化能力(记忆进化 + 回归护栏),把"它怎么越用越强、又不退化"讲清楚;② 给这篇综述的 awesome-list 提 PR(补一篇归类)或基于它做一个小型 self-evolving demo。能讲清"自进化的取舍 + 安全"本身就是很硬的认知深度。
