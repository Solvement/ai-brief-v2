"多 agent 协作"是你做 agent 产品迟早要碰的硬骨头,而 MetaGPT 是这条线上**最被广泛抄作业的工程范式**(68.5k★)。它给你的不是某个模型,而是一套可直接套用的答案:**怎么让一群 agent 分工而不互相带偏**——用 SOP 约束流程、用结构化产物代替闲聊、用消息池+订阅协作、用"真跑代码"的可执行反馈纠错。你做任何"多步骤、多角色"的 AI 工作流,这四招都能直接迁移。

| --- | --- | --- |

## 系统设计心法

一句话:**多 agent 不是"多请几个 LLM 聊天",是"把人类成熟流程 + 结构化产物 + 运行反馈"工程化。**
- **流程 > 自由对话**:让 agent 照 SOP 办事、产出可验收的中间工件,从源头压住级联幻觉。
- **结构 > 大白话**:角色间传 schema 化文档,歧义低、可机器校验、可订阅。
- **运行 > 评审**:能真跑就别只靠 LLM 互评——可执行反馈是质量的最大来源。
- **订阅 > 广播**:每个角色只拿自己关心的,防信息过载(这也是省 token 的关键)。

## 作品集角度(Portfolio angle)

"多 agent 工程"是当下极好讲的方向。两条路都硬:① 自己搭一个小型 role-based 多 agent 流水线(带消息池 + 可执行反馈),把"agent 不互相带偏"讲清楚;② 给 `FoundationAgents/MetaGPT`(68.5k★)提一个真实 PR(新角色/新 Action/新数据源)——**给头部 agent 框架 merged PR** 是很硬的简历点。
