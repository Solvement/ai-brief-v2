## 为什么这对我重要

你早晚要给某个垂直领域(医疗、金融、教育、工业)做一个"能看懂专业内容"的 AI 应用。那时你最缺的**从来不是模型,而是数据**——没有高质量、真贴合任务、且不含捷径的训练/评测数据,再强的底座也调不出可靠的领域能力。这篇恰恰是一份**可照抄的"领域数据工厂"蓝图**:怎么从零定知识范围、怎么把知识点变成能搜到真实素材的查询、怎么用模型放量又不被单模型偏见带歪、怎么造一个**逼模型真干活、不让它走捷径的评测集**。它教你的是 agent/数据时代最值钱的一类工程能力:**造数据,而不只是调模型**。

> [!key] 和我们正在做的东西直接同源
> 我们这个站的语料(L0 学习底座)本质也是一条**数据生产流水线**:发现→筛选→深读→质检→入库。VideoKR 的几个内核可以**直接搬进我们的流水线**:① **七模型池 + 人验准入门槛**(每步抽样评错、错误率达标才准入)= 我们"哪个模型配跑哪个深读阶段"的客观选型协议;② **视频依赖过滤 = 我们的"去捷径"思想**(一条深读结论,如果不看原文也能写出来,那它就是水的);③ **CoT 推理链由独立强模型验证 = 我们的冷审深读门**(生成器≠评判器)。**你学 VideoKR,等于在学怎么把我们自己的数据流水线做得更硬。**

## 系统设计心法

一句话:**模型是借来的,数据是你的护城河——把"造数据"当成一等工程,而不是训练前的杂活。**
- **数据设计 > 算法花活**:本文用最普通的 SFT→GRPO 就把小模型推过同量级 SOTA,证明很多"涨点"其实是数据红利。先把数据做对,再谈调参。
- **永远先查"捷径"**:你的数据/评测里若混着"不干活也能对"的题,你测出来的"能力"是假的。上线前给数据做一遍捷径探测,是最便宜的防自欺。
- **生成器≠评判器**:模型造的数据要用**独立的**模型/专家来验;同一个模型自产自验,系统性偏见验不出来。七模型池 + 人验门槛就是这个思想的工程化。
- **质量门要量化、可接受残噪**:不是追求零噪声(做不到),而是**测出残噪率、和某个可接受基线对比**(本文用"和人工种子错误率相当"当标尺)。
- **天下没有免费午餐**:专精数据会牺牲通用能力(本文通用视频推理普遍微跌)。设计数据配比时就要想清楚你要专才还是通才。

## 作品集角度(Portfolio angle)

这能撑起一个 **"垂直领域数据工厂 / 去捷径评测"** 方向的硬作品——这正是当下最缺、最难外包、最能体现"懂数据"的方向:不是又一个 RAG demo,而是"**给一个专业领域从零造出能训能评的数据**"的基建。两条路都值钱:① 自己挑一个小垂直领域,跑通一条 mini 版 VideoKR 流水线(哪怕只做文本/图片,不做视频);② 做一个**捷径探测器 + 去捷径评测集**作为通用工具,任何数据集丢进去都能报出"水分率"。

## 简历 / 面试故事

诚实写法:

- *"Built a knowledge-driven synthetic data pipeline for a vertical domain: knowledge-bank to scenario-grounded retrieval to skill-tiered QA generation to self-consistency / shortcut / CoT filtering."*
- *"Designed a multi-model qualification protocol — sampled 100 real inputs per pipeline step, had experts label errors, and admitted a model to a step only below a fixed error threshold — to reduce single-model bias in synthetic data."*
- *"Built a shortcut-probing evaluation harness that removes QA examples answerable from a single frame / text-only, exposing inflated 'video reasoning' accuracy in existing benchmarks."*

> [!key] 面试可讲
> "你怎么给一个专业领域造出可靠的训练数据?"——用本文论证:**先建知识范围、把知识点场景化去采真实素材、按技能分层生成、再用'去捷径过滤 + 独立验证'卡质量**;并点出两个生产级坑:① 别让一个模型既造又验(系统性偏见);② 专精数据会牺牲通用能力(没有免费午餐)。体现的是数据思维 + 实验纪律。

## 学习清单

**必学**
- 合成数据流水线五步骨架(知识库→场景化采集→技能分层生成→多重过滤→切分)
- 去捷径思想:单帧/文本可解题就是水题,数据和评测都要先剔
- 生成器≠评判器:多模型池 + 人验准入门槛

**有用**
- data-centric 实验纪律:锁死管线只动数据,让涨点干净归因
- 质量门量化:测残噪率 + 设可接受基线,而非追零噪声

**可选**
- CoT 监督的取舍:对知识密集有用、对通用可能拖累 → 按目标配数据
- 合成数据的成本结构(7 万美元推理 + 专家时薪)→ 何时值得自造数据 vs 复用
