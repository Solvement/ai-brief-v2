## 为什么这对我重要

你搭多 agent 系统时,最累、最玄学的就是"**拓扑**"——谁调谁、谁先谁后、加几个 agent。GPTSwarm 给了一个反直觉但很强的答案:**别手搭,把它变成可优化的图,让系统自己学连边、自己剪掉坏队友。** 这把"agent 编排"从手艺活升级成"可学习对象",是 agent 基建里很前沿、也很值钱的一类能力。对你想做的"越用越好的 agent"尤其关键。

> [!key] 和我们北极星(L1 自进化)同源
> "优化 agent 图的连边/提示"**本质就是结构层面的自进化**。我们的 L1(BriefEvolve)要让流水线自己变好——GPTSwarm 的"在 DAG 概率分布上用 REINFORCE 学拓扑 + 剪有害边"正是一张可借的现成蓝图。

## 系统设计心法

一句话:**把 agent 编排当"可优化的图",而不是手搭的流程图。**
- **结构即参数**:节点提示和节点间连边都是图上参数,能优化就别手调。
- **学拓扑 > 加 agent**:实验证明"连对边"比"连更多边"更重要——别堆 agent,学连接。
- **剪枝是鲁棒性**:让优化器自动断开坏队友,比人工挑队友更稳。
- **离散→连续**:把"选不选这条边"松弛成概率,才能用梯度优化——这是把很多离散结构问题变可学的通用招。

## 作品集角度(Portfolio angle)

"自优化的 agent 编排"是非常硬核、好讲的方向。两条路:① 自己做一个 mini agent-graph optimizer(节点提示 + 边拓扑都能优化,带剪枝 demo),把"系统自己学连边"讲清楚;② 给 `metauto-ai/GPTSwarm` 提 PR(新环境/新优化器/新 agent 类型)。
