# Career & Learning · 求职与成长

## 为什么这对我重要

你要做的是 **agent 产品**,而 agent 产品上线后第一杀手就是**不可靠**——它会自信地走错、还一路把错的当真。这篇恰恰把"**agent 为什么错、错在哪、怎么定位**"做成了可操作的工程框架。它教你的不是某个模型,而是 agent 时代最值钱的一类能力:**给 agent 建过程级的可靠性诊断与护栏**。

> [!key] 和我们正在做的东西直接同源
> 我们这个站的**冷审深读门**(作者写 → 独立冷审查证 → 逐维补差 → 收敛)本质就是一个 **Agent-as-a-Judge / DRIFT 式审计器**:把"分析"当 agent 轨迹,把每个数字/结论当 claim,去原文里查 support。**你学 DRIFT,等于在学怎么把我们自己的质量门做得更硬**——这就是"你学 + AI 自己也学"的同一块料。

## 系统设计心法

一句话:**把 agent 的"中间承诺"当一等公民来治理,而不是只盯最终输出。**
- **过程 > 结果**:可靠性问题几乎都藏在中间没核实的 claim 里;只看终答=放弃了所有可诊断信号。
- **区分探索与承诺**:agent 试错很正常,真正危险的是"把没证据的东西当既定前提往下用"。护栏要卡在"承诺"那一刻。
- **轨迹内一致 ≠ 真实为真**:DRIFT 的盲区(只查 agent 自己看到的证据)正是你做生产系统要补的——高影响 claim 要接**外部 verifier 重新取证**。
- **错误会传播**:定位**首错**比"哪都可能错"有用一百倍;设计时就要留住"谁依赖谁"的链路。

## 作品集角度(Portfolio angle)

这能撑起一个 **"可靠 agent / agent 评测"** 方向的硬作品——这正是当下最缺、最好讲的方向:不是又一个 chatbot,而是"**让 agent 别乱来**"的基建。两条路都值钱:① 自己造一个过程级 agent 审计/护栏小系统;② 给 `NJU-LINK/DRIFT` 这类活跃项目提一个真实 PR(外部 verifier / 在线版 / 新数据源解析)。

## 可造(Build ideas)

| 造什么 | 范围 | 时间 | 为什么有用 |
| --- | --- | --- | --- |
| Online-DRIFT 护栏 | agent 每形成一个关键 claim 就实时查 support,弱/缺则拦在进最终答案前 | 1-2 周 | 把"事后审计"变成"事前拦截",是真能上产品的护栏 |
| External-Verifier 层 | 在 DRIFT 后接一个独立检索器重核高影响证据(补"轨迹内一致≠真"盲区) | 1 周 | 直击论文最大局限,讲故事极顺 |
| Claim-aware RAG | 把 RAG 从 query→retrieve→answer 改成 query→retrieve→起草claims→逐条查证→answer | 4-7 天 | 把 DRIFT 思想用在**生成**而非审计,降幻觉 |
| Process Reward Model | 用 TELBench 的 span 标签训一个 per-span 错误概率打分器 | 1-2 周 | 比每次跑多阶段 LLM 审计更便宜,可做在线 reward |

## 简历 / 面试故事

诚实写法:

- *"Built an online claim-verification guardrail for a deep-research agent: tracks consequential claims, checks evidence support (direct/weak/missing/conflicting), and blocks unsupported commitments before they reach the final answer."*
- *"Built a process-level agent eval harness (span-level F1 + first-error accuracy) on real long-horizon trajectories, showing errors concentrate in decision/finalization, not retrieval."*
- *"(若做了 PR)Contributed an external-evidence verifier to NJU-LINK/DRIFT, addressing its trajectory-internal-consistency blind spot."*

> [!key] 面试可讲
> "你怎么让一个会多步推理的 agent 别越错越远?"——用本文论证:**别只看最终答案,要在'承诺'那一刻查证据、把护栏放在 decision/finalization**;并点出"轨迹内一致≠真实为真"这个生产级坑。体现的是过程思维 + 系统思维。
