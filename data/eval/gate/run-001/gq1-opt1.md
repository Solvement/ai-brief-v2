**结论先行：做成"混合控制器"——先把记忆操作做成显式可调用的带类型动作（typed action），把可学习策略留到后面，别一上来就上强化学习。**

这条路上有两组互相打架的真实证据，看清楚再动手：

**可学习派的最强证据**：有一套方法（AgeMem）把六个记忆操作——长期记忆的 增/改/删 + 短期上下文的 取回/压缩/过滤——做成 agent 能自己调用的动作，再用强化学习端到端训练它"何时用哪个"。在两个模型、五个长程任务上，均分超过 Mem0、A-Mem 这类模块化记忆基线（作者自报，受控基准）。关键是它的消融：把强化学习那一阶段去掉，分数明显回落（自报约降 8–9 个百分点）——说明真正承重的是"学习"，不是那个统一接口本身。而且给旧基线补上同样的模块也追不上，作者据此说"统一策略 ≠ 模块叠加"。

**固定规则派的最强反证**：一个零训练、纯靠写死规则与权重的检索系统（三路 BM25+向量+实体图用 RRF 融合），自报在 LongMemEval-S 上 R@5 命中率 95.2%（注意：这是检索召回，不含更新/遗忘/冲突维度，且作者明确只有这个数是自测、其余是他方自报）。说明在纯检索这一维，规则式仍能打，而且有可控、低成本、可解释、不依赖训练或外部 API 的天然优势。

**所以两边都没赢透**：它们根本没在同一个 benchmark 上比过；学习派的训练轨迹全来自单一数据源（HotpotQA），跨域泛化作者自己没验证。

**给你的落地动作**：
1. **先把"何时存/取/改/删"从写死的 if-else 抽成一组带类型的动作**（Add/Update/Delete/Retrieve/Summary/Filter）——让这些决策显式、可观测、可替换。这一步无门槛、立刻能做，也是上面那套方法里唯一不需要训练就能借的部分。
2. **检索骨架直接抄成熟配方**：多路检索（BM25+向量）用 RRF 融合，并做"缺腿降级"（某条通道挂了权重归零、其余重归一化），保证不会整体失忆。
3. **强化学习那一层先别碰**——没有可训练模型就是过度工程化。把后端结构预留成"未来可挂学习层"即可。等你攒够真实任务反馈、确实看到规则瓶颈时，再把最痛的那个决策点（通常是取用排序）换成可学习策略。

一句话：规则层守安全边界与可解释性，学习层管动态优化；接口先行，训练后置。
