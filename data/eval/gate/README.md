# 判决实验 · Gate Experiment（库 vs 直接问模型）

**问题（Kevin 2026-06-13）**：Mind Palace 的对象库，在真实 need 时刻，是否胜过"直接问前沿模型"？
这是整个 ROS 方向的承重假设。逻辑成立不够，必须实测（research-loop：先把 benchmark 做强）。

## 协议
- **臂 A — 模型直答**：同一个模型，只给问题，凭自身知识答（= 没有库的基线）。
- **臂 C — 模型 + 对象库**：同一个模型，问题 + 该问题相关的对象库内容（claims/mechanisms/propositions 人话裁决）。
- **臂 B — 模型 + 原文全文**：塞相关论文全文。*（pilot 暂缓：记忆类多为 repo 源、无干净 arxiv 全文；先打 A vs C 这把关键刀。）*
- **盲评**：臂用随机标签（opt-X）写盘，key 单独存、judge 看不到。独立冷 agent 排名（generator≠critic）。
- **判据（应用构建者视角，非研究者）**：可操作（告诉我该做什么）/ 可追溯（结论挂得到来源）/ 前沿感知 / 对不确定性诚实 / **"别重复造轮子"——指得到已有最优解**。

## 反过拟合（Kevin 硬约束）
问题只是例子、非固定集；要测**泛化**（对没见过的新问题也能答），不许把这几题背下来当范式。
焦点 = AI 应用 / 用好工具搭 agent，不是研发新机制。

## 判决
- **C 赢不了 A** → 对象抽取层没证成：砍层，回去堆解读。
- **C 赢 B（后续）** → 结构化优于堆全文：抽取层站得住，继续投。
- 同时看：C 在**语料覆盖的问题**上赢多少、在**语料薄/没覆盖**的问题上是否拖后腿（暴露边界）。

## 现状
- run-001：pilot 2 题（均落在语料强项=记忆簇），验证 harness + 取首个信号。
- 下一轮：加入语料**薄**的问题（如 Agentic RAG / 预测引擎选型）测边界 + 扩到 10–20 题测泛化。
