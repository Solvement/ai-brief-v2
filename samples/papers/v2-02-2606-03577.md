# 一个 8B 模型，在跨视角匹配上做到 70.5 F1

**一句话钩子**

论文报告，Qwen3-VL-8B 经过 DCRL 训练后，在 ReasonMatch-Bench 上达到 70.5 F1。[^1]

**元信息**

论文：Eliciting Complex Spatial Reasoning in MLLMs through Wide-Baseline Matching

作者：Hao Zhong、Muzhi Zhu、Shenyan Zeng、Anzhou Li、Cong Chen、Hua Geng、Duochao Shi、Wentao Ye、Tao Lin、Hao Chen、Chunhua Shen

机构：浙江大学、蚂蚁集团、西湖大学

类型：benchmark + 数据生成管线 + 可验证奖励强化学习

arXiv：2606.03577

**开篇张力**

多模态大模型已经很会看图说话，但把同一个场景从两个相差很大的角度拍下来，让模型判断两张图里哪些标注点指向同一个物理位置，问题就变了。它不再只是识别椅子、墙面、架子，而是要在脑中维护一个稳定的三维场景：这个角落换了视角还在不在？这个点是同一层货架，还是被遮住了？

这正是 wide-baseline matching（宽基线匹配，指两张图拍摄视角差异很大时仍要找出同一物理位置）想测的能力。

**一句话主张**

这篇论文的核心主张是：宽基线匹配可以把 MLLM 的空间推理变成一种可验证、可规模化、可训练的任务；而 Dynamic Correspondence Reinforcement Learning（动态对应强化学习，简称 DCRL）比单纯监督模仿更能激发跨视角几何能力。

**结果先看**

论文报告，Qwen3-VL-8B-Instruct 原始模型在 ReasonMatch-Bench 上只有 27.5 F1，DCRL 后达到 70.5 F1；同一评测中，GPT-5-mini 为 57.9 F1，GPT-5-Chat 为 51.5 F1，Qwen3-VL-235B 为 49.2 F1，Gemini-2.5-Pro 为 42.8 F1。

这意味着，在这个特定空间对应任务上，一个专门训练过的 8B 开源模型，超过了更大的开源模型和多种闭源模型。更反直觉的是，这不是靠人工写好的 CoT 监督，而是靠最终匹配是否正确的可验证奖励。[^2]

但它远未到人类水平。在最大视角差异的 90 个样本子集上，非专家人类为 84.0 F1，DCRL 为 52.0 F1，GPT-5-mini 为 37.2 F1。也就是说，DCRL 明显提升了模型，但人类仍保留巨大优势。

**看点先读**

- 论文把空间推理从开放式问答变成了可自动判分的物理对应题：两个视角里的标注区域必须对上同一个三维位置。
- 数据管线的关键不是人工写解释，而是从 RGB-D 视频和 SfM 重建中抽取几何可验证的监督信号。
- DCRL 的核心不是让模型背答案，而是按视角跨度、候选点复杂度和空间分布逐步加难。

**为什么宽基线匹配比看图问答更难？**

普通图像问答经常可以靠局部线索过关：看到木纹，就说桌子；看到白色大平面，就说墙。但宽基线匹配要求模型判断两个不同视角中的标注点是否对应同一个物理位置。这里会同时出现视角旋转、尺度变化、遮挡、重复结构和细粒度差异。

论文把任务设计成文本驱动的 correspondence mapping（对应关系映射）。模型看到两张带编号点的图片，输出从图 A 点位到图 B 点位或 none 的 JSON。公式里，模型预测的是 f_hat: {1,...,n} -> {1,...,m} ∪ {∅}。人话翻译就是：A 图里的每个点，要么在 B 图里找到同一个物理位置，要么承认它在另一个视角里没有可靠对应。

可以把它想成在两个监控摄像头之间找同一处货架标签。不能只说这里像白墙、那里像木板；你必须知道摄像头转过去以后，左侧、深处、遮挡关系和物体层级如何变化。

**ReasonMatch-Bench 怎样把难题做成可验证题？**

ReasonMatch-Bench 不是让人手工随便圈点。论文先从 RGB-D 数据和 SfM 重建中拿到真实几何对应。对 RGB-D 数据，做法是把第一张图的像素根据深度反投影到三维，再投到第二张图，并检查深度一致性和颜色一致性。对 SfM 数据，则使用两个视图共享的三维 landmark。

这样得到的是能被几何验证的对应关系，而不是含糊的文字标签。之后，论文再把密集匹配点过滤成适合视觉提示的点集，并加入 distractor（干扰点）和 none 选项，让任务更接近真实跨视角场景。

评测集从 220k-pair corpus 中抽取 2,810 image pairs；场景组成是 indoor 55.1%、object 28.0%、outdoor 16.9%；任务难度分布是 L1 32.5%、L2 36.8%、L3 30.7%。这些数字说明，论文试图避免评测只偏向某一种场景或某一种简单匹配形式。

所以这对读者意味着什么：这个 benchmark 测的不是模型会不会描述图像，而是它能否把描述落到机器可检查的空间对应表上。

**DCRL 到底训练了什么？**

DCRL 可以类比成训练一个人看不同摄像头的回放。先让他在相近视角里找同一把椅子，再让他在更远视角、更多遮挡、更多干扰点里找同一块区域。难度不是一次性拉满，而是跟着模型表现逐步推进。

奖励函数由两部分组成：format compliance（格式合规，输出能被解析）和 matching correctness（匹配正确率，每个点的对应或 none 判断是否正确）。公式 r = w_f · r_format + w_m · r_match 的人话意思是：答案既要写成机器能读的格式，也要真的把点匹配对。

训练实现上，论文报告使用 GRPO，group size G = 32，每次更新的有效 batch 是 16 × 32 trajectories，KL 系数 β = 0.005，学习率为 10^-6。课程方面，图像级别按 10 个 overlap bins 推进，平均 accuracy reward 在 20 个 training steps 的滑动窗口中超过 0.8 后进入更难视角。

点级别也在加难。L1 是无干扰的一一匹配；L2 是目标图有干扰点的选择性匹配；L3 是两张图都有干扰点和不可见区域的部分匹配。附录报告，L1 使用 3-5 个 matchable points 且没有 distractors；L2 使用 1-2 个 matchable points 和 3-6 个 target distractors；L3 使用 3-6 个 matchable points，并在两张图各加入 3-6 个 distractors。

**主结果：训练过的 8B 模型超过强基线，但物体细粒度仍难**

ReasonMatch-Bench 总体上，DCRL 为 70.5 F1，对比 Qwen3-VL-8B-Instruct 的 27.5 F1，是同骨干大幅提升。对比其他模型，GPT-5-mini 为 57.9 F1，GPT-5-Chat 为 51.5 F1，Qwen3-VL-235B 为 49.2 F1，Gemini-2.5-Pro 为 42.8 F1，Claude-4.5-Sonnet 为 41.7 F1。

更细看，Object L3 是最难区域。DCRL 在 Object L3 上为 33.7，对比 GPT-5-mini 的 27.8、Qwen3-VL-235B 的 22.1、Gemini-2.5-Pro 的 19.0，仍有优势，但绝对分数不高。这说明物体中心场景缺少室内和室外场景中的大尺度环境锚点，模型更难靠全局布局定位同一物理点。

人类研究进一步校准了难度。在 90 个最大视角差异样本上，Human 为 84.0 F1，DCRL 为 52.0 F1，GPT-5-mini 为 37.2 F1。uCO3D 物体中心子集尤其困难：Human 为 62.1 F1，DCRL 为 27.8 F1。

所以这不是一个“模型已经懂三维”的结论，而是一个更克制的结论：可验证的宽基线训练能显著推进 MLLM 的跨视角对应能力，但细粒度物体几何仍是硬骨头。

**迁移结果：它有没有只背 ReasonMatch？**

如果 DCRL 只是练熟了 ReasonMatch 的答案格式，那么外部空间 benchmark 不应该稳定受益。论文报告，OmniSpatial 从 43.60 提升到 48.87；MindCube 从 40.01 到 43.52；SAT Real 从 70.00 到 75.30。

通用视觉理解上也没有报告退化：MME-RealWorld 从 62.8 到 63.8，MMStar 从 59.8 到 62.5，RealWorldQA 从 69.5 到 70.5，V*Bench 从 84.8 到 85.9。

这对应用构建者很关键。空间专训经常有一个风险：模型会变得只会做某个格式化任务，损伤原有视觉语言能力。至少在论文报告的这些评测上，DCRL 没有出现这种退化。

**消融回答：是机制有效，还是只是训练数据更多？**

最关键的对照是 RL vs SFT。SFT 在 ReasonMatch 上从 base 的 27.5 到 51.0，确实有效；但 DCRL 到 70.5，比 SFT 高 19.5 点。更有意思的是 SAT：base 是 70.0，SFT 掉到 41.3，而 DCRL 到 75.3，比 SFT 高 34.0 点。[^3]

这说明，监督模仿可以让模型学会训练集里的对应形式，但可能牺牲迁移；可验证奖励和探索式训练更可能保留并强化可迁移空间能力。

课程学习的消融也支持这一点。Dynamic Curriculum 为 70.5，No Curriculum 为 65.3，Easy-only 为 59.9，Hard-only 为 62.3。也就是说，动态课程比均匀采样高 5.2 点，比只训简单样本高 10.6 点，比只训困难样本高 8.2 点。

这组数字回答了一个常见质疑：不是只要用 RL 训一下就行，难度如何组织同样重要。先建立基础对应能力，再逐渐增加视角差、干扰点和细粒度空间分布，效果优于把所有样本混在一起。

**这项工作对工程和方法论意味着什么？**

工程上，它给 AI 应用提供了一种训练空间能力的现实路径：用已有视频三维数据构造可判分任务，而不是等待昂贵人工标注。对于机器人、巡检、室内导航、货架理解、远程协作、AR 辅助等应用，系统经常要处理“同一物理位置在不同视角下在哪里”这个问题，ReasonMatch 的任务形式很贴近这些需求。

方法论上，它把空间推理拆成了几个可控旋钮：视角跨度、候选点数量、干扰点、遮挡、空间分布。这样 benchmark 不只给一个总分，还能解释模型到底是卡在局部纹理、全局布局、none 校准，还是 reasoning-to-answer 的结构化输出。

应用构建者还应注意一点：论文中的 prompt 明确要求模型用场景级 landmark、三维相对关系和遮挡变化推理，而不是只描述像素位置。这提示我们，做多图视觉代理时，输入格式和输出约束本身就是产品能力的一部分。

**局限**

第一，模型距离人类仍远。最大视角差异子集上，DCRL 是 52.0 F1，人类是 84.0 F1；物体中心 uCO3D 上，DCRL 是 27.8 F1，人类是 62.1 F1。

第二，论文主要研究 pairwise cross-view matching。真正完整的空间智能还需要多视图联合推理、时间动态、三维场景理解和语义知识的整合。

第三，overlap score 是 source-aware 的难度分层工具，不是跨数据来源完全可比的统一物理难度标尺。

第四，训练算力成本、不同 GRPO group size、不同 reward 权重、不同数据规模的系统消融，论文未提供。

**一句话结语**

这篇论文最有价值的地方，不是宣布 MLLM 已经理解三维世界，而是给出了一个能自动验证、能逐步加难、能暴露真实短板的空间推理训练场。
